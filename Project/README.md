# Проектная работа

Проектная работа — интегральный КИМ, позволяющий проверить несколько компетенций и индикаторов в условиях комплексной практической задачи.

### 1. Общие сведения
**Раздел:** Проект  
**Вид КИМ:** защита индивидуального проекта  
**Максимальный балл:** 100  
**Назначение:** оценка результатов выполнения проектного задания, качества программной реализации, корректности применения инструментов разработки и уровня владения материалом при защите <br>
**Контрольно-измерительные материалы (КИМ):** индивидуальное проектное задание и его защита. Оценивание проводится по 100-балльной шкале по критериям, отражающим постановку задачи, выбор модели машинного обучения, выбор соответствующих данных (датасета), архитектуру решения, реализацию backend на Python, работу с данными, развертывание модели машинного обучения, применение инструментов промышленной разработки для развёртывания моделей машинного обучения, качество кода и уровень защиты проекта.

### 3. Объект оценивания
Оценивается индивидуальный проект в виде клиент-серверного приложения, обеспечивающего развертывание и использование модели машинного обучения, работу с данными, обработку запросов, документирование и реализация работы с API, обоснование выбранного решения и защиту решения.

### 4. Содержание проекта
**Проект должен включать:**
- постановку задачи, решаемой при помощи средств машинного обучения (рекомендуемый список моделей и задач приведен в [п.7](#7-перечень-моделей-машинного-обучения-и-возможные-области-их-применения));
- поиск данных (датасет) для выбранной задачи;
- описание архитектуры клиент-серверного приложения;
- реализацию backend-сервиса на Python;
- поиск, выбор и интеграция работы приложения с данными (датасетом);
- API для загрузки и обработки данных;
- обоснование выбора модели машинного обучения (логистическая регрессия, дерево решений, случайный лес, градиентный бустинг и т.д.), применительно к задаче;
- API для применения модели машинного обучения;
- получение метрик качества и/или параметров модели;
- обработку ошибок и исключений;
- документацию API;
- подготовку и защиту результатов проекта.

### 6. Шкала оценивания
- **86–100 баллов** — отлично.
- **71–85 баллов** — хорошо.
- **56–70 баллов** — удовлетворительно.
- **0–55 баллов** — неудовлетворительно.

### 7. Перечень моделей машинного обучения и возможные области их применения
| Модель | Возможный датасет | Задача | Описание применения |
|---|---|---|---|
| Логистическая регрессия | Табличные данные о заявках с числовыми и категориальными признаками. | Классификация результата обработки заявки. | Подходит для базовой классификации на структурированных данных; быстро обучается, легко внедряется в API и дает интерпретируемый результат. |
| Дерево решений | Сведения о пользователях или клиентах: возраст, активность, число обращений, статус. | Прогнозирование класса объекта по табличным признакам. | Удобно для наглядного описания логики принятия решения; модель легко интерпретировать и использовать в учебном проекте. |
| Метод опорных векторов (SVM) | Набор измерений или характеристик объектов с несколькими классами. | Многоклассовая классификация. | Подходит для задач, где классы разделяются в пространстве признаков; уместна для сравнения с другими алгоритмами в проекте. |
| Наивный Байесовский классификатор | Небольшой набор структурированных признаков с меткой класса. | Классификация объектов по заданным признакам. | Быстрая базовая модель, удобная для первого сравнения и демонстрации работы вероятностного подхода. |
| Случайный лес | Табличные данные о клиентах, заявках или событиях. | Бинарная классификация или прогнозирование исхода. | Ансамблевый метод, который обычно дает устойчивое качество и подходит для практических задач на структурированных данных. |

### 8. Об использовании технологий генеративного искусственного интеллекта при выполнении проекта

Использование генеративного искусственного интеллекта допускается в ограниченном объеме (в рамках допустимого в конкретном образовательном учреждении), исключительно для вспомогательных и подготовительных задач, не связанных с подменой самостоятельного выполнения проекта, и только по согласованию с преподавателем. Применение ИИ не освобождает обучающегося от необходимости самостоятельно выполнять анализ, проектирование, разработку, проверку работоспособности и защиту проекта. При защите проекта обучающийся должен уметь объяснить все использованные решения без опоры на внешние источники.

*Разрешается применять ИИ, например, для:*
- уточнения формулировок и языковой редактуры текста;
- поиска идей для структуры проекта или интерфейса;
- получения пояснений по документации и типовым подходам;
- помощи в составлении черновиков технического описания или инструкции.

*Не допускается использование ИИ для:*
- полного выполнения проектной работы вместо обучающегося;
- написания кода основной части проекта без последующего самостоятельного понимания и доработки;
- генерации ответов на вопросы защиты без личного владения материалом;
- подмены анализа, проектных решений и выводов обучающегося;
- использования с нарушением требований преподавателя или локальных нормативных актов.

*При использовании ИИ обучающийся должен:*
- указывать факт его применения;
- быть готовым объяснить и обосновать все использованные материалы и решения;
- при необходимости предоставить преподавателю перечень задач, для которых применялся ИИ.
