## Модель 1. Логистическая регрессия

### Возможная задача
**Классификация результата обработки заявки**

### Краткое описание задачи
Логистическую регрессию можно использовать для бинарной классификации исхода заявки по набору табличных признаков. Например, модель по данным о клиенте, типе обращения, сумме, канале подачи и других характеристиках определяет, будет ли заявка **одобрена** или **отклонена**. Такая задача хорошо подходит для демонстрации базовой классификации на структурированных данных, поскольку модель быстро обучается, легко интерпретируется и дает понятный результат.

### Краткое описание датасета
**Home Credit Default Risk** — открытый датасет с большим числом табличных признаков о клиентах и их кредитных заявках; содержит числовые и категориальные поля, пригодные для бинарной классификации результата обращения, а также реальные данные с пропусками, которые требуют предобработки.  
Ссылка: [Kaggle — Home Credit Default Risk](https://www.kaggle.com/c/home-credit-default-risk/data)

### Порядок подготовки модели для внедрения в проект
1. Загрузить данные и выделить целевую переменную.
2. Проверить типы признаков, состав выборки и наличие пропусков.
3. Выполнить очистку данных, обработку пропусков и кодирование категориальных признаков.
4. Разделить данные на обучающую и тестовую выборки.
5. Обучить модель логистической регрессии.
6. Оценить качество модели по метрикам классификации.
7. Сохранить обученную модель и все необходимые преобразования.
8. Подготовить входной формат для предсказания по одной записи.
9. Подготовить формат массового предсказания по CSV-файлу, если он требуется.
10. Проверить воспроизводимость обучения и предсказания на новых данных.

### Что можно предсказывать
- одобрение или отклонение заявки;
- вероятность отказа;
- принадлежность заявки к одному из двух классов по заданным признакам.

## Модель 2. Дерево решений

### Возможная задача
**Прогнозирование класса клиента по поведенческим и демографическим признакам**

### Краткое описание задачи
Дерево решений можно применять для классификации клиентов по набору простых признаков: возраст, активность, число обращений, статус, длительность взаимодействия и другие параметры. Модель позволяет наглядно показать логику принятия решения и подходит для задач, где важно объяснить, почему объект отнесен к тому или иному классу.

### Краткое описание датасета
**Bank Marketing** — открытый датасет с данными о маркетинговых кампаниях банка; включает демографические, поведенческие и кампанийные признаки, а целевая переменная показывает, оформил ли клиент срочный депозит. Данные содержат как числовые, так и категориальные поля, удобные для построения дерева решений.  
Ссылка: [UCI Machine Learning Repository — Bank Marketing](https://archive.ics.uci.edu/dataset/222/bank+marketing)

### Порядок подготовки модели для внедрения в проект
1. Загрузить данные и определить целевой класс.
2. Проверить распределение признаков и категорий.
3. Обработать пропуски и закодировать категориальные признаки.
4. Разделить данные на обучающую и тестовую выборки.
5. Обучить дерево решений.
6. Настроить базовые гиперпараметры при необходимости.
7. Оценить качество модели и интерпретируемость правил.
8. Сохранить модель и схему предобработки.
9. Подготовить формат входных данных для одного объекта и, при необходимости, для CSV.
10. Проверить корректность предсказаний на новых данных.

### Что можно предсказывать
- согласится клиент на предложение или нет;
- отнесется ли клиент к положительному классу;
- будет ли обращение успешным.

## Модель 3. Метод опорных векторов (SVM)

### Возможная задача
**Многоклассовая классификация объектов по измерениям признаков**

### Краткое описание задачи
Метод опорных векторов можно использовать для классификации объектов, если признаки образуют разделимые или частично разделимые классы в пространстве характеристик. Такая задача удобна для демонстрации многоклассовой классификации и сравнения SVM с другими алгоритмами на одном и том же датасете.

### Краткое описание датасета
**Iris** — компактный классический датасет с измерениями цветков ириса; содержит четыре числовых признака и три класса, что делает его удобным для демонстрации многоклассовой классификации и работы метода опорных векторов.  
Ссылка: [UCI Machine Learning Repository — Iris](https://archive.ics.uci.edu/dataset/53/iris)

### Порядок подготовки модели для внедрения в проект
1. Загрузить данные и проверить структуру признаков.
2. Убедиться, что все признаки числовые и не требуют сложного кодирования.
3. При необходимости выполнить масштабирование признаков.
4. Разделить данные на обучающую и тестовую выборки.
5. Обучить SVM-модель.
6. Подобрать ядро и параметры модели.
7. Оценить качество классификации по нескольким классам.
8. Сохранить модель и шаги предобработки.
9. Подготовить входной формат для одного объекта.
10. Проверить корректность работы модели на новых данных.

### Что можно предсказывать
- вид цветка по измерениям чашелистиков и лепестков;
- принадлежность объекта к одному из нескольких классов;
- класс образца по набору числовых признаков.

Особенно удобно использовать его для простых текстовых или табличных данных, например для определения категории сообщения, типа обращения или тематической принадлежности.

### Краткое описание датасета
**SMS Spam Collection** — набор коротких текстовых сообщений с двумя классами, где нужно отличить спам от обычных сообщений; датасет удобен для демонстрации вероятностной текстовой классификации и базовой предобработки текста.

2. Очистить текст, привести его к единому виду.
3. Выполнить токенизацию и векторизацию признаков.
4. Разделить данные на обучающую и тестовую выборки.
5. Обучить наивный Байесовский классификатор.
6. Оценить качество по метрикам классификации.
7. Сохранить модель и текстовый векторизатор.
8. Подготовить формат ввода для одного сообщения.
9. Подготовить массовую обработку текстов, если требуется.
10. Проверить работу модели на новых сообщениях.

### Что можно предсказывать
- спам / не спам;
- категория сообщения;
- принадлежность текста к одному из двух классов.

### Краткое описание датасета
**Breast Cancer Wisconsin (Diagnostic)** — набор числовых признаков, вычисленных по изображениям биопсии; содержит 30 характеристик формы и структуры ядер клеток, а целевая переменная показывает, является ли опухоль доброкачественной или злокачественной. Датасет хорошо подходит для FCNN из-за полностью числового представления данных.  
Ссылка: [UCI Machine Learning Repository — Breast Cancer Wisconsin (Diagnostic)](https://archive.ics.uci.edu/dataset/17/breast+cancer+wisconsin+diagnostic)

3. Выполнить нормализацию или стандартизацию.

7. Оценить качество модели.
8. Сохранить обученную сеть и схему предобработки.

10. Проверить работу модели на новых данных.

### Что можно предсказывать
- доброкачественная или злокачественная опухоль;
- класс объекта по набору числовых признаков;
- принадлежность к одному из двух классов.

**Классификация изображений по визуальным признакам**

### Краткое описание датасета
**Fashion-MNIST** — открытый набор из 70 000 изображений одежды и аксессуаров размером 28×28 в оттенках серого; содержит 10 классов и хорошо подходит для демонстрации свёрточных сетей в задаче многоклассовой классификации.  
Ссылка: [Fashion-MNIST](https://github.com/zalandoresearch/fashion-mnist)

3. Нормализовать пиксели и подготовить входной формат.
4. Разделить данные на обучающую и тестовую выборки.
5. Построить и обучить CNN.
6. При необходимости применить аугментацию данных.
7. Оценить качество классификации.
8. Сохранить модель и параметры предобработки.
9. Подготовить формат предсказания для одного изображения.
10. Проверить предсказание на новых изображениях.

### Что можно предсказывать
- тип одежды по изображению;
- класс визуального объекта;
- принадлежность изображения к одной из категорий.
