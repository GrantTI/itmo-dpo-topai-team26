### Детализация добавленных занятий

#### Занятие 1.7 (СРС, 4 ч.) — Самостоятельное изучение нотации C4

**Задание для студентов:**
1. Изучить нотацию C4 (Context, Container, Component, Code)
2. На основе лекционного материала (Занятие 1.1) нарисовать C4-диаграммы для своего проекта:
   - **Уровень 1 (Context):** Пользователь ↔ Веб-приложение
   - **Уровень 2 (Container):** Frontend (Vue.js + Nginx) ↔ Backend (FastAPI)
   - **Уровень 3 (Component):** Контроллеры, сервисы, утилиты в бэкенде
3. Инструменты: PlantUML, Draw.io, Mermaid или Lucidchart
4. Результат: 3 диаграммы, добавленные в репозиторий проекта (папка `/docs`)

---

#### Занятие 3.3 (ЛР, 6 ч.) — Расширенная работа с Nginx и Docker

**Содержание:**
1. **Nginx как reverse-proxy:**
   - Настройка проксирования запросов на бэкенд (`proxy_pass`)
   - Настройка заголовков (`Host`, `X-Real-IP`, `X-Forwarded-For`)
   - Настройка WebSocket (если нужно)

2. **Балансировка нагрузки (опционально):**
   - Настройка upstream для нескольких экземпляров бэкенда
   - Стратегии балансировки (round-robin, least-conn)

3. **Кэширование статики:**
   - Настройка `expires` и `Cache-Control` для статических файлов
   - Инвалидация кэша через хеши в именах файлов

4. **Multi-stage сборка Docker:**
   - Оптимизация размера образов
   - Разделение на build-stage и production-stage

5. **Docker Compose в production:**
   - Использование `.env` файлов
   - Настройка `restart: always`
   - Управление volumes для логов и данных

**Практическое задание:**
- Собрать проект с оптимизированными Docker-образами
- Настроить Nginx с кэшированием статики
- Запустить проект через `docker-compose up -d`
- Проверить работу приложения на `http://localhost`

```
backend/
├── app/
│   ├── api/

│   │   │   ├── __init__.py
│   │   │   ├── data.py                # /api/data/*

│   │   ├── data_service.py            # Работа с данными (CSV, Pandas)
│   │   ├── model_service.py           # Работа с моделями (scikit-learn)
│   │   └── validation_service.py      # Hold-out валидация
│   ├── core/                          # Ядро приложения
│   │   ├── __init__.py
│   │   ├── config.py                  # Настройки (переменные окружения)

│       ├── data_processor.py          # Предобработка данных
│       └── validators.py              # Валидация входных данных
├── requirements.txt
├── Dockerfile
└── .env.example
```
