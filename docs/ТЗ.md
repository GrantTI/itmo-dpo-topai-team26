# Техническое задание на учебный проект
## "Технологии разработки и развертывания моделей машинного обучения в клиент-серверных приложениях”"

---

## 1. Общие сведения

### 1.1. Наименование проекта
Веб-приложение для внедрения и визуализации моделей машинного обучения

### 1.2. Цель проекта
Создать простой веб-интерфейс для практического изучения и экспериментов с моделями машинного обучения, позволяющий выполнять полный цикл работы: загрузка данных, обучение моделей, визуализация результатов и экспорт.

### 1.3. Целевая аудитория
Студенты, изучающие машинное обучение, и специалисты, желающие быстро протестировать линейные модели на своих данных.

## 3. Функциональные требования

### 3.1. Выбор модели

**Описание**: Пользователь может выбрать одну из нескольких предустановленных линейных моделей.

**Доступные модели**:
1. Linear Regression (Ordinary Least Squares)
2. Ridge Regression (L2-регуляризация)
3. Lasso Regression (L1-регуляризация)
4. ElasticNet (комбинация L1 и L2)
5. SGDRegressor (стохастический градиентный спуск)

**Интерфейс**:
- Выпадающий список (dropdown) или группа радиокнопок для выбора модели
- Отображение краткого описания выбранной модели
- Поля для ввода гиперпараметров (зависят от модели):
  - Linear Regression: без гиперпараметров
  - Ridge: параметр alpha (регуляризация)
  - Lasso: параметр alpha
  - ElasticNet: параметры alpha и l1_ratio
  - SGDRegressor: learning_rate, max_iter, alpha

### 3.2. Загрузка данных

**Описание**: Пользователь загружает CSV-файл с данными для обучения модели.

**Требования**:
- Загрузка через drag-and-drop или выбор файла
- Поддержка CSV формата с разделителями: запятая, точка с запятой, табуляция
- Автоматическое определение типов данных (числовые, категориальные)
- Предпросмотр загруженных данных (первые 5-10 строк)
- Отображение статистики: количество строк, столбцов, типы данных
- Автоматическое определение целевой переменной (последний столбец по умолчанию)
- Возможность выбрать столбец как целевую переменную из выпадающего списка
- Валидация данных: проверка на пропуски, выбросы (опционально)

**Интерфейс**:
- Кнопка "Загрузить данные" и зона для drag-and-drop
- Таблица с предпросмотром данных
- Выпадающий список для выбора целевой переменной
- Чекбокс "Стандартизировать данные" (StandardScaler)

### 3.3. Обучение модели

**Описание**: Запуск процесса обучения на загруженных данных с выбранными параметрами.

**Функционал**:
- Кнопка "Обучить модель" (активна только после загрузки данных)
- Индикация процесса обучения (progress bar или спиннер)
- Отображение времени обучения
- Вывод метрик качества:
  - R² (коэффициент детерминации)
  - MSE (Mean Squared Error)
  - RMSE (Root Mean Squared Error)
  - MAE (Mean Absolute Error)
- Отображение коэффициентов модели (weights) в виде таблицы

**Интерфейс**:
- Кнопка запуска обучения
- Блок с метриками (карточки или таблица)
- Таблица коэффициентов модели

### 3.4. Кросс-валидация и графики ошибок

**Описание**: Визуализация результатов кросс-валидации для оценки стабильности модели.

**Функционал**:
- Разбиение train/test (hold-out валидация)
- Выбор метрики для оценки: MSE, R², MAE
- Отображение графика ошибок по фолдам:
  - Box plot или bar chart с ошибками для каждого фолда
  - Линия среднего значения и доверительный интервал
- Отображение графика "Actual vs Predicted" (фактические vs предсказанные значения)
- Отображение графика "Residuals" (остатки)

**Интерфейс**:
- Вкладка или секция "Кросс-валидация"
- Выпадающие списки для выбора количества фолдов и метрики
- Блок с графиками

### 3.5. Прогнозирование

**Описание**: Загрузка новых данных (тестового набора) и построение прогнозов.

**Функционал**:
- Загрузка CSV-файла для прогнозирования (с той же структурой, что и обучающий)
- Отображение предсказанных значений
- Отображение графика "Предсказанные vs Фактические" (если есть целевая переменная)
- Отображение графика "График ошибок прогноза" (разница между предсказанными и фактическими)

**Интерфейс**:
- Секция "Прогнозирование"
- Кнопка загрузки файла для прогноза
- Таблица с результатами прогноза
- Блок с графиками

### 3.6. Экспорт результатов

**Описание**: Экспорт графиков и данных в различных форматах.

**Функционал**:
- Экспорт графиков в форматах:
  - PNG (растровое изображение)
  - SVG (векторное изображение)
  - PDF (документ)
- Экспорт данных в CSV:
  - Коэффициенты модели
  - Результаты кросс-валидации (ошибки по фолдам)
  - Прогнозы для тестовых данных
- Кнопка "Скачать все" (архив с графиками и CSV)

**Интерфейс**:
- Кнопки экспорта рядом с каждым графиком
- Выпадающий список формата при экспорте
- Кнопка "Экспортировать все" в общем меню

### 5.2. API Endpoints

**Модели**:
- `GET /api/models` - список доступных моделей с параметрами
- `GET /api/models/{model_id}` - информация о конкретной модели

**Данные**:
- `POST /api/data/upload` - загрузка CSV файла
- `GET /api/data/preview` - предпросмотр данных
- `POST /api/data/process` - обработка данных (выбор целевой переменной)

**Обучение**:
- `POST /api/train` - обучение модели
- `GET /api/train/status/{job_id}` - статус обучения (асинхронно)
- `GET /api/train/metrics` - получение метрик

**Кросс-валидация**:
- `POST /api/cross-validation` - запуск кросс-валидации
- `GET /api/cross-validation/plots` - получение графиков

**Прогнозирование**:
- `POST /api/predict` - загрузка данных и построение прогнозов
- `GET /api/predict/results` - получение результатов прогнозирования

**Экспорт**:
- `GET /api/export/plot/{plot_id}` - экспорт графика
- `GET /api/export/data/{data_type}` - экспорт данных в CSV
- `POST /api/export/all` - экспорт всех результатов (архив)

## 6. Docker Compose конфигурация

### 6.1. docker-compose.yml

```yaml
version: '3.8'

services:
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: ml-backend
    ports:
      - "8000:8000"
    environment:
      - PYTHONUNBUFFERED=1
      - ENVIRONMENT=development
    volumes:
      - ./backend:/app
      - ./data:/app/data
      - ./logs:/app/logs
      - ./exports:/app/exports
    networks:
      - ml-network
    restart: unless-stopped

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: ml-frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    environment:
      - VITE_API_URL=http://backend:8000
    networks:
      - ml-network
    restart: unless-stopped

networks:
  ml-network:
    driver: bridge

volumes:
  data:
  logs:
  exports:
```

### 6.2. Backend Dockerfile

```dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000", "--reload"]
```

### 6.3. Frontend Dockerfile

```dockerfile
FROM node:18-alpine as build-stage

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

FROM nginx:stable-alpine as production-stage

COPY --from=build-stage /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### 6.4. Frontend nginx.conf

```nginx
server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /static {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```
