# Лабораторная работа
## Тема: Развёртывание клиент-серверного приложения с моделями машинного обучения в контейнерной среде

**Цель работы:** Сформировать умение настраивать и кастомизировать контур развёртывания клиент-серверного приложения с моделями машинного обучения: контейнеризация сервисов, оркестрация средствами Docker Compose, конфигурация через переменные окружения и публикация через Nginx.

**Продолжительность:** 6 академических часов

**Связь с занятием:** 3.4

**Проверяемые результаты обучения**

| Компетенция | Индикатор | Уровень | Дескриптор в контексте задания |
|---|---|---|---|
| LC-5. Способен применять и (или) проектировать инструменты и инженерные практики промышленной разработки, развёртывания, эксплуатации и мониторинга систем ИИ | LC-5.1 | С | Настраивает и кастомизирует Docker, Docker Compose и базовую публикацию через Nginx для рабочей версии клиент-серверного приложения с моделями машинного обучения; обеспечивает воспроизводимый запуск и проверку доступности сервисов |

**Необходимые условия:** работоспособные серверная (модуль 1) и клиентская (модуль 2) части учебного проекта; установленные Docker и Docker Compose; текстовый редактор. Рекомендуется выполненная лабораторная работа занятия 3.3.

**Объект выполнения.** Лабораторная работа выполняется на учебном (эталонном) составе проекта дисциплины либо на согласованном с преподавателем учебном стенде. Полный перенос и документирование контура на итоговом репозитории обучающегося относятся к самостоятельной работе занятия 3.6.

**Форма контроля:** текущая (защита лабораторной работы).

---

## Задание 1. Подготовка контейнерных образов сервисов приложения

**Описание.** Подготовить контейнерные образы для серверной и клиентской частей учебного приложения.

**Требования:**

1. Для серверной и клиентской частей созданы отдельные Dockerfile (либо согласованная с преподавателем эквивалентная схема).
2. Образы собираются без ошибок.
3. В образах отсутствуют файлы с реальными секретами; зависимости устанавливаются воспроизводимо (фиксированный перечень в манифесте зависимостей).

---

## Задание 2. Оркестрация сервисов средствами Docker Compose

**Описание.** Описать и запустить многосервисный контур средствами Docker Compose.

**Требования:**

1. В файле `docker-compose.yml` (либо эквиваленте) описаны сервисы серверной и клиентской частей приложения.
2. Настроена общая сеть Compose; взаимодействие сервисов осуществляется по именам сервисов, а не исключительно через `localhost` хоста разработки.
3. Заданы публикация портов и базовые зависимости порядка запуска (например, `depends_on`, где это уместно).
4. Контур запускается одной командой Compose и корректно останавливается.

---

## Задание 3. Включение сервиса инференса в контейнерный контур

**Описание.** Включить в контур учебный сервис инференса, подготовленный в рамках занятия 3.3 (либо эквивалентную заготовку преподавателя).

**Требования:**

1. Сервис инференса описан в Compose и доступен другим сервисам по имени в сети Compose.
2. Серверная часть обращается к сервису инференса с использованием адреса из переменных окружения, согласованного с сетью Compose.
3. Демонстрируется успешный вызов инференса через развёрнутый контур.

*При отсутствии наработок занятия 3.3 допускается использование предоставленной преподавателем заготовки; отсутствие сервиса инференса в контуре снижает оценку по соответствующему критерию.*

---

## Задание 4. Конфигурация параметров среды

**Описание.** Вынести параметры среды из исходного кода и жёстко заданных значений в конфигурацию окружения.

**Требования:**

1. Параметры (адреса сервисов, порты, режимы работы, секреты при наличии) задаются через переменные окружения и/или файл `env_file`.
2. В репозитории присутствует файл `.env.example` (либо аналог) без реальных секретов.
3. Изменение параметра окружения влияет на поведение контура без изменения исходного кода приложения.

---

## Задание 5. Публикация через Nginx и проверка контура

**Описание.** Настроить базовую публикацию приложения через Nginx и подтвердить работоспособность развёрнутой версии.

**Требования:**

1. В контуре присутствует сервис Nginx (отдельный контейнер либо согласованный вариант).
2. Настроена маршрутизация к клиентской части и/или к программному интерфейсу серверной части (обратный прокси на уровне базовой рабочей схемы).
3. После запуска контура демонстрируются: открытие клиентского интерфейса и успешный ответ ключевого метода программного интерфейса (например, проверка состояния, перечень моделей или предсказание — в зависимости от состава проекта).
4. Обучающийся поясняет назначение сервисов в Compose и путь запроса от клиента к серверной части.

*Углублённое конфигурирование обратного прокси-сервера (кэширование, балансировка нагрузки, детальная маршрутизация) и многостадийная сборка образов отрабатываются в лабораторной работе занятия 3.5.*

---

## Порядок выполнения и сдачи

1. Подготовить Dockerfile и файл оркестрации Compose.
2. Включить сервис инференса в контур (задание 3).
3. Настроить переменные окружения и файл `.env.example`.
4. Запустить контур, продемонстрировать клиентский интерфейс и программный интерфейс.
5. Продемонстрировать останов и повторный запуск контура.
6. Кратко пояснить схему сервисов и роль Nginx.

**Формат сдачи:** файлы контейнеризации в репозитории проекта; устная защита с демонстрацией запущенного контура.

## Критерии оценивания

Максимум: **15 баллов**. Работа считается зачтённой при результате не ниже **9 баллов**.

| Критерий | Баллы | Показатели |
|---|:---:|---|
| Сборка образов | 0–3 | Dockerfile для сервисов; образы собираются |
| Оркестрация сервисов | 0–3 | Несколько сервисов; сеть Compose; воспроизводимый запуск и останов |
| Сервис инференса в контуре | 0–2 | Сервис включён в Compose; доступен по сети; вызов через контур успешен |
| Конфигурация окружения | 0–3 | Переменные окружения; `.env.example`; отсутствие секретов в коде и в системе контроля версий |
| Публикация и проверка контура | 0–2 | Базовый обратный прокси; клиентский интерфейс и программный интерфейс доступны |
| Обоснование решения | 0–2 | На защите поясняются состав контура и путь запроса |

## Правила использования внешних материалов и генеративного ИИ

- Допускается использование официальной документации Docker, Docker Compose и Nginx, а также учебных шаблонов, предоставленных преподавателем.
- Допускается применение генеративного ИИ для справок и подготовки черновиков конфигурации при условии, что обучающийся понимает назначение директив и способен изменить конфигурацию на защите.
- Не допускается сдача непроверенной конфигурации, которую обучающийся не может пояснить.
- Секреты и ключи доступа в репозиторий не включаются.
