# Лабораторная работа
## Тема: Развёртывание ML-приложения в production-контуре

**Цель работы:** Сформировать умение настраивать и кастомизировать контур развёртывания клиент-серверного ML-приложения: контейнеризация сервисов, оркестрация средствами Docker Compose, конфигурация через переменные окружения и публикация через Nginx.

**Продолжительность:** 6 академических часов

**Связь с занятием:** 3.4

**Проверяемые результаты обучения**

| Компетенция | Индикатор | Уровень | Дескриптор в контексте задания |
|---|---|---|---|
| LC-5. Способен применять и (или) проектировать инструменты и инженерные практики промышленной разработки, развёртывания, эксплуатации и мониторинга систем ИИ | LC-5.1 | С | Настраивает и кастомизирует Docker, Docker Compose и базовую публикацию через Nginx для рабочей версии клиент-серверного ML-приложения; обеспечивает воспроизводимый запуск и проверку доступности сервисов |

**Необходимые условия:** работоспособные backend (модуль 1) и frontend (модуль 2) учебного проекта; установленные Docker и Docker Compose; текстовый редактор. Рекомендуется выполненная лабораторная работа занятия 3.3 (при включении inference/mock-сервиса в контур).

**Форма контроля:** текущая (защита лабораторной работы).

---

## Задание 1. Dockerfile для сервисов приложения

**Описание.** Подготовьте контейнерные образы для backend и frontend учебного ML-приложения.

**Требования:**

1. Для backend и frontend созданы отдельные `Dockerfile` (или согласованная с преподавателем эквивалентная схема).
2. Образы собираются командой сборки без ошибок.
3. В образах не размещаются файлы с реальными секретами; зависимости устанавливаются воспроизводимо (фиксированный перечень в манифесте зависимостей).

---

## Задание 2. Оркестрация сервисов в Docker Compose

**Описание.** Опишите и запустите многосервисный контур средствами Docker Compose.

**Требования:**

1. В `docker-compose.yml` (или эквиваленте) описаны сервисы backend и frontend; при наличии — сервис inference/mock из занятия 3.3.
2. Настроена общая сеть Compose; сервисы обращаются друг к другу по именам сервисов, а не только через `localhost` хоста.
3. Прописаны проброс портов и базовые зависимости запуска (например, `depends_on`, где уместно).
4. Контур поднимается одной командой запуска Compose и останавливается штатно.

---

## Задание 3. Конфигурация через переменные окружения

**Описание.** Вынесите параметры среды из кода и жёстко заданных значений в конфигурацию окружения.

**Требования:**

1. Параметры (URL сервисов, порты, режимы, секреты при наличии) задаются через переменные окружения и/или `env_file`.
2. В репозитории присутствует `.env.example` (или аналог) без реальных секретов.
3. Изменение параметра окружения влияет на поведение контура без правки исходного кода приложения.

---

## Задание 4. Публикация через Nginx и проверка контура

**Описание.** Настройте базовую публикацию приложения через Nginx и подтвердите работоспособность развёрнутой версии.

**Требования:**

1. В контуре присутствует сервис Nginx (отдельный контейнер или согласованный вариант).
2. Настроена маршрутизация к frontend и/или backend (reverse-proxy на уровне базовой рабочей схемы).
3. После запуска контура демонстрируются: открытие клиентского интерфейса и успешный ответ ключевого API (например, health, список моделей или predict — по составу проекта).
4. Студент поясняет назначение сервисов в Compose и путь запроса от клиента до backend.

*Углублённая настройка Nginx (кэширование, балансировка, тонкая маршрутизация) и multi-stage сборка отрабатываются в лабораторной работе занятия 3.5.*

---

## Порядок выполнения и сдачи

1. Подготовить Dockerfile и `docker-compose.yml`.
2. Настроить переменные окружения и `.env.example`.
3. Запустить контур, продемонстрировать UI и API.
4. Показать остановку и повторный запуск контура.
5. Кратко объяснить схему сервисов и роль Nginx.

**Формат сдачи:** файлы контейнеризации в репозитории проекта; устная защита с демонстрацией запущенного контура.

## Критерии оценивания

Максимум: **15 баллов**. Работа считается зачтённой при результате не ниже **9 баллов**.

| Критерий | Баллы | Показатели |
|---|:---:|---|
| Сборка образов | 0–3 | Dockerfile для сервисов; образы собираются |
| Compose и сеть сервисов | 0–4 | Несколько сервисов; сеть Compose; воспроизводимый up/down |
| Конфигурация окружения | 0–3 | env / env_file; `.env.example`; без секретов в коде и git |
| Nginx и проверка контура | 0–3 | Базовый reverse-proxy; UI и API доступны после запуска |
| Понимание решения | 0–2 | На защите объясняет состав контура и путь запроса |

---

## Правила использования внешних материалов и генеративного ИИ

- Допускается использование официальной документации Docker, Docker Compose и Nginx, а также учебных шаблонов преподавателя.
- Допускается применение генеративного ИИ для справок и черновиков конфигурации при условии, что студент понимает назначение директив и может изменить конфигурацию на защите.
- Запрещается сдавать непроверенную конфигурацию, которую студент не может объяснить.
- Секреты и ключи доступа в репозиторий не включаются.
