# Модуль 3. Интеграция и развёртывание

Директория содержит описания контрольно-измерительных материалов (КИМ) модуля 3 дисциплины «Технологии разработки и развёртывания моделей машинного обучения в клиент-серверных приложениях».

## Назначение модуля

Модуль направлен на формирование умений интегрировать клиентскую и серверную части приложения (включая применение модели машинного обучения через API серверной части) и развёртывать приложение в контейнерной среде с использованием средств контейнеризации и обратного прокси-сервера.

**Объём:** 26 а.ч. (лекции — 4; лабораторные работы — 14; самостоятельная работа — 8).

## Компетенции и уровни

| Код | Название компетенции | Индикатор | Уровень |
|---|---|---|---|
| PL-1 | Способен применять язык программирования Python для решения задач в области ИИ | PL-1.1 | С |
| LC-5 | Способен применять и (или) проектировать различные инструменты и инженерные практики промышленной разработки, развёртывания, эксплуатации и мониторинга систем ИИ | LC-5.1 | С |

**Дескрипторы в контексте модуля.** По завершении модуля обучающийся:

- согласовывает контракт REST API между клиентом и сервером, организует HTTP-взаимодействие и настраивает политику CORS и обратный прокси на уровне интеграции компонентов;
- обеспечивает в интегрированном контуре вызов и применение модели машинного обучения через API серверной части (результат модуля 1; рабочий паттерн — внутрипроцессный инференс);
- настраивает и кастомизирует контур развёртывания клиент-серверного приложения с моделями машинного обучения (Docker, Docker Compose, Nginx, переменные окружения, многостадийная сборка), обеспечивает воспроизводимый запуск рабочей версии программного обеспечения и проверку работоспособности сервисов.

## Место в структуре дисциплины

Модуль опирается на результаты модулей 1 (Backend, REST API, применение моделей через API) и 2 (Frontend, интерфейсы взаимодействия с API) и обеспечивает подготовку развёрнутого контура для тестирования и защиты проекта в модуле 4.

Занятие 3.2 знакомит с паттернами размещения инференса (внутрипроцессный, выделенный сервис, внешний API). Обязательная реализация выделенного или внешнего сервиса инференса в лабораторных работах модуля **не требуется**.

Занятие 3.6 закрепляет навыки развёртывания на материале собственного проекта дисциплины и оценивает воспроизводимый контейнерный контур (Docker, Docker Compose, Nginx, переменные окружения). Итоговая проектная работа и защита в модуле 4 оценивают целостность продукта (функциональность, тестирование, презентация) на уже развёрнутом контуре; критерии занятия 3.6 не дублируют интегральную оценку проекта.

Дальнейшее углубление практик жизненного цикла систем ИИ (оркестрация контейнеров, непрерывная интеграция и развёртывание, расширенное сопровождение моделей в продуктивной среде) относится к дисциплинам блока MLOps / DevOps.
