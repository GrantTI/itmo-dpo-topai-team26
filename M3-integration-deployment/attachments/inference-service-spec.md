# Спецификация учебного сервиса инференса

Учебный имитатор сервиса инференса предназначен для лабораторных работ занятий 3.3–3.4 модуля «Интеграция и развёртывание». Сервис не реализует обучение моделей; он принимает запрос и возвращает детерминированный учебный ответ.

## Назначение

Обеспечить воспроизводимую интеграцию серверного приложения на FastAPI с отдельным HTTP-сервисом инференса (паттерн внешнего или выделенного сервиса).

## Контракт программного интерфейса

| Параметр | Значение |
|---|---|
| Метод | `POST` |
| Путь | `/predict` |
| Формат | `application/json` |
| Базовый URL (локально) | `http://127.0.0.1:8100` |
| Базовый URL (в сети Compose) | `http://inference:8100` (имя сервиса может быть согласовано в `docker-compose.yml`) |

### Тело запроса

```json
{
  "features": [1.0, 2.0, 3.0]
}
```

| Поле | Тип | Описание |
|---|---|---|
| `features` | массив чисел | Признаки объекта для учебного предсказания; длина не менее 1 |

### Тело успешного ответа (`200 OK`)

```json
{
  "prediction": 6.0,
  "model_id": "dummy-sum-v1"
}
```

| Поле | Тип | Описание |
|---|---|---|
| `prediction` | число | Учебный результат; для заготовки допускается сумма элементов `features` |
| `model_id` | строка | Идентификатор учебной модели |

### Ответы об ошибках

| Код | Условие |
|---|---|
| `422` | Некорректное тело запроса (отсутствует `features` или неверный тип) |
| `500` | Внутренняя ошибка имитатора (для проверки обработки сбоев на стороне клиента допускается учебный режим отказа по заголовку или флагу окружения) |

## Параметры окружения клиента (основное приложение)

| Переменная | Пример | Назначение |
|---|---|---|
| `INFERENCE_BASE_URL` | `http://127.0.0.1:8100` | Базовый адрес сервиса инференса |
| `INFERENCE_TIMEOUT_SEC` | `3` | Предельное время ожидания ответа |
| `INFERENCE_API_TOKEN` | (пусто в учебной заготовке) | Необязательный токен; при наличии передаётся в заголовке `Authorization: Bearer …` |

## Требования к реализации заготовки

1. Сервис запускается независимо от основного приложения.
2. Поведение при корректном запросе детерминировано и документировано.
3. Для проверки устойчивости клиентского кода имитатор должен позволять смоделировать недоступность (останов процесса) без изменения кода основного приложения.
