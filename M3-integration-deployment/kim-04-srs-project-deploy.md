# Самостоятельная работа
## Тема: Контейнеризация и запуск собственного проекта в production-режиме

**Цель работы:** Закрепить навыки настройки и кастомизации production-контура на материале собственного проекта дисциплины: подготовить образы, описать Compose и Nginx, обеспечить воспроизводимый запуск и проверку работоспособности сервисов.

**Продолжительность:** 8 академических часов

**Связь с занятием:** 3.6

**Проверяемые результаты обучения**

| Компетенция | Индикатор | Уровень | Дескриптор в контексте задания |
|---|---|---|---|
| LC-5. Способен применять и (или) проектировать инструменты и инженерные практики промышленной разработки, развёртывания, эксплуатации и мониторинга систем ИИ | LC-5.1 | С | Самостоятельно настраивает и кастомизирует контур развёртывания своего клиент-серверного ML-приложения; обеспечивает воспроизводимый запуск рабочей версии ПО и демонстрирует доступность сервисов |

**Необходимые условия:** выполненные лабораторные работы занятий 3.4 и 3.5 (или эквивалентный контур); собственный проект дисциплины с backend и frontend; Docker и Docker Compose.

**Форма контроля:** рубежная (сдача контура развёртывания перед переходом к тестированию и защите в модуле 4).

**Граница с итоговой проектной работой.** В рамках данной СРС оценивается только воспроизводимый production-контур (контейнеры, Compose, окружение, Nginx, проверка доступности). Полнота ML-функциональности, качество интерфейса, тестовое покрытие и презентация оцениваются в проектной работе / модуле 4.

---

## Задание 1. Контур развёртывания собственного проекта

**Описание.** Перенесите и адаптируйте практики занятий 3.4–3.5 на репозиторий собственного проекта.

**Требования:**

1. В проекте есть Dockerfile для backend и frontend (допускается multi-stage, если уже освоено в 3.5).
2. Описан `docker-compose.yml` с сетью сервисов; сервисы обращаются друг к другу по именам Compose.
3. Параметры среды вынесены в переменные окружения; в репозитории есть `.env.example` без секретов.
4. Настроен Nginx для публикации UI и/или API.

---

## Задание 2. Воспроизводимый запуск и проверка

**Описание.** Обеспечьте запуск контура «с нуля» и подтвердите работоспособность.

**Требования:**

1. Контур поднимается документированной последовательностью команд (краткая инструкция `README` раздела deploy или аналог: build / up).
2. После запуска доступны клиентский интерфейс и ключевой API проекта (по составу проекта: health, данные, модели или predict).
3. Выполняются штатные остановка и повторный запуск без ручного «лечения» контейнеров.
4. Секреты отсутствуют в истории сдаваемых файлов.

---

## Задание 3. Фиксация состава контура

**Описание.** Кратко зафиксируйте архитектуру развёртывания для передачи в модуль 4.

**Требования:**

1. Перечень сервисов Compose и их назначение (backend, frontend, Nginx, при наличии — inference/mock).
2. Указаны порты публикации и основные переменные окружения (без значений секретов).
3. Описан путь запроса: клиент → Nginx → backend (и далее к модели/инференсу — по факту проекта).

---

## Порядок выполнения и сдачи

1. Адаптировать контейнеризацию под свой проект.
2. Поднять контур, провести проверку UI и API.
3. Подготовить краткое описание состава контура.
4. Сдать материалы преподавателю (рубежный контроль модуля 3).

**Формат сдачи:** репозиторий проекта с файлами развёртывания; демонстрация запуска; краткое описание контура (в README или отдельном md).

---

## Критерии оценивания

Максимум: **15 баллов**. Работа считается зачтённой при результате не ниже **9 баллов**.

| Критерий | Баллы | Показатели |
|---|:---:|---|
| Полнота контура | 0–5 | Dockerfile, Compose, env-пример, Nginx для своего проекта |
| Воспроизводимый запуск | 0–5 | up/down; UI и API доступны; инструкция запуска |
| Описание контура | 0–3 | Сервисы, порты, путь запроса |
| Аккуратность конфигурации | 0–2 | Нет секретов в git; понятная структура файлов |

*Не оцениваются в этой СРС:* качество обучения моделей, дизайн UI, покрытие тестами, презентация к защите.

---

## Правила использования внешних материалов и генеративного ИИ

- Допускается опора на собственные наработки лабораторных работ 3.4–3.5 и официальную документацию Docker / Nginx.
- Допускается применение генеративного ИИ для справки при условии понимания и способности воспроизвести конфигурацию на сдаче.
- Запрещается включать секреты в репозиторий и сдавать необъяснимую конфигурацию.
