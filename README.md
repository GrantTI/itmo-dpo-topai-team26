# Фонд оценочных средств по дисциплине «Технологии развертывания моделей машинного обучения в клиент-серверных приложениях»

> **Статус:** шаблон репозитория. Замените все маркеры `[ЗАПОЛНИТЬ]` и удалите учебные примеры перед публикацией.

Этот репозиторий предназначен для открытого и воспроизводимого представления фонда оценочных средств (ФОС) по дисциплине «Технологии развертывания моделей машинного обучения в клиент-серверных приложениях». Здесь объединены рабочая программа дисциплины, модель измерения результатов обучения, контрольно-измерительные материалы (КИМ), критерии оценивания, методические рекомендации и используемые образовательные ресурсы. Результаты обучения связываются с компетентностно-ролевой моделью в области искусственного интеллекта (КРМ версии 3.0).

## 1. О дисциплине

**Цель дисциплины.** [ЗАПОЛНИТЬ: 2–3 предложения о том, чему и зачем обучаются слушатели/студенты.]

**Основные задачи.** [ЗАПОЛНИТЬ: перечислите ключевые учебные задачи без чрезмерной детализации.]

**Структура и содержание.** Дисциплина состоит из 4 модулей: "Backend (REST API)", "Frontend", "Интеграция и развертывание", "защита проекта". Обучение включает лекции / лабораторные работы / проект / самостоятельную работу и завершается дифференцированным зачетом.

**Пререквизиты и связи с другими дисциплинами.** 

### 1. Объектно-ориентированное программирование (ООП)
*   **Код компетенций**: PL-2, PL-3
*   **Уровень**: Средний
*   **Что требуется знать**:
    *   Принципы ООП (наследование, полиморфизм, инкапсуляция, абстракция).
    *   Паттерны проектирования (базовое понимание).
    *   Умение проектировать структуру классов и взаимодействие между объектами.

---

### 2. Обработка данных на Python
*   **Код компетенций**: PL-1
*   **Уровень**: Базовый
*   **Что требуется знать**:
    *   Синтаксис Python и работа со встроенными типами данных.
    *   Библиотеки для анализа данных: `Pandas` и `NumPy`.
    *   Навыки чтения/записи данных из файлов (CSV, JSON, Excel).

---

### 3. Машинное обучение
*   **Код компетенций**: ML-3, ML-4, ML-7, MF-4, BD-1, LC-5
*   **Уровень**: Базовый
*   **Что требуется знать**:
    *   **ML-3 / ML-4**: Основные алгоритмы классификации и регрессии.
    *   **ML-7**: Методы оценки качества моделей (метрики).
    *   **MF-4**: Инженерия признаков (Feature Engineering) и предобработка данных.
    *   **BD-1**: Работа с большими данными (базовые концепции).
    *   **LC-5**: Линейная алгебра и математическая статистика (базовый уровень).

---

### 4. MLOps & DevOps
*   **Код компетенций**: LC-4
*   **Уровень**: Средний
*   **Что требуется знать**:
    *   Базовые принципы CI/CD (непрерывная интеграция и доставка).
    *   Работа с системами контроля версий (Git).
    *   Контейнеризация (Docker) — базовое понимание.
    *   Автоматизация развертывания моделей в продакшн.

---

### 5. Технологии тестирования программного обеспечения
*   **Код компетенций**: PL-3
*   **Уровень**: Средний
*   **Что требуется знать**:
    *   Виды тестирования (модульное, интеграционное, функциональное).
    *   Фреймворки для тестирования на Python (`pytest`, `unittest`).
    *   Написание Unit-тестов для проверки корректности кода.

---

### 6. Компьютерные сети
*   **Код компетенций**: LC-5
*   **Уровень**: Базовый
*   **Что требуется знать**:
    *   Сетевая модель OSI и TCP/IP (базовые уровни).
    *   Понимание работы HTTP/HTTPS протоколов.
    *   Основы работы с сокетами и API-запросами.

**Планируемые результаты обучения.** По завершении дисциплины обучающийся сможет [ЗАПОЛНИТЬ: 3–6 проверяемых результатов обучения, сформулированных через наблюдаемые действия].

Полное описание приведено в [РПД](docs/rpd.md).

## 7. Порядок заполнения

1. Заполните РПД и сформулируйте проверяемые результаты обучения.
2. Выберите релевантные профессиональные роли и компетенции КРМ.
3. Для каждого результата выберите индикатор и требуемый уровень.
4. Заполните модель измерения.
5. Создайте КИМ и рубрики оценивания по шаблонам.
6. Согласуйте систему итогового балла.
7. Добавьте ресурсы, сведения о команде и лицензию.
