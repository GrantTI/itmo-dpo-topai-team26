## 2. Подготовка к выполнению

### Теоретический минимум

Перед выполнением работы убедитесь, что вы знаете:

#### Основы нотации C4 (Уровень 1 и 2)

**Уровень 1 (System Context):**
- Показывает систему в целом и ее взаимодействие с пользователями и внешними системами
- Элементы: акторы (человечки), системы (прямоугольники), взаимодействия (стрелки)

**Уровень 2 (Container):**
- Показывает контейнеры (приложения, базы данных, серверы)
- Каждый контейнер — это отдельный исполняемый процесс
- Элементы: контейнеры с указанием технологии, связи с протоколами

#### Типовая веб-архитектура

```
Пользователь → Nginx (прокси/статику) → Backend (FastAPI) → Database (PostgreSQL)
```

#### Сетевые протоколы и порты

| Протокол | Порт | Назначение |
|----------|------|------------|
| HTTP | 80 | Нешифрованный веб-трафик |
| HTTPS | 443 | Шифрованный веб-трафик |
| HTTP (внутренний) | 8000 | Внутреннее API (FastAPI) |
| TCP | 5432 | База данных PostgreSQL |
| TCP | 6379 | Redis |

#### Распределение ролей в архитектуре

- **Nginx:** маршрутизатор, прокси-сервер, отдает статику, балансирует нагрузку
- **FastAPI:** бизнес-логика, работа с БД, обработка запросов
- **PostgreSQL:** хранилище данных (stateful-сервис)
- **Vue.js/React:** статические файлы (артефакты, не контейнеры)

### Анализ задания

**Ключевые требования ко всем заданиям:**

1. **Правильное использование нотации C4:**
   - Акторы — человечки
   - Контейнеры — прямоугольники с названием и технологией
   - Связи — стрелки с протоколом, портом и направлением

2. **Архитектурные принципы:**
   - Только бэкенд работает с БД
   - Nginx — только прокси и статика
   - Фронтенд — только статика, отдается через Nginx
   - Нет прямых связей "фронтенд → БД" и "Nginx → БД"

3. **Документирование:**
   - Все стрелки должны быть подписаны (протокол, порт, направление)
   - Должна быть легенда (для цветов, типов линий)

4. **Конкретика:**
   - Указывать версии технологий (например, FastAPI 0.100, PostgreSQL 15)
   - Указывать конкретные протоколы и порты
