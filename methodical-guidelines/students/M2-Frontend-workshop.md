# Методические рекомендации по выполнению воркшопа «Визуализация графиков для ML моделей: ECharts vs Chart.js»

## 1. Общие положения

Настоящие методические рекомендации предназначены для организации работы студентов в рамках воркшопа по визуализации данных для моделей машинного обучения. Воркшоп предполагает выполнение практических заданий по созданию интерактивных графиков с использованием двух библиотек визуализации — Chart.js и ECharts — в экосистеме Vue.js 3. Формат работы предусматривает разделение на две команды, каждая из которых осваивает одну из библиотек, что позволяет провести сравнительный анализ их возможностей и производительности.

Целями воркшопа являются:

- освоение методов интеграции библиотек визуализации Chart.js и ECharts в проекты на Vue.js 3;
- приобретение навыков построения различных типов графиков для сравнения прогнозных и фактических данных;
- изучение подходов к визуализации результатов работы алгоритмов машинного обучения;
- формирование компетенций в области разработки интерактивных дашбордов.

Воркшоп рассчитан на студентов, владеющих основами JavaScript и Vue.js 3 на уровне, достаточном для создания компонентов и работы с реактивными данными.

## 2. Теоретическая подготовка

### 2.1. Выбор библиотеки визуализации

Выбор библиотеки для визуализации данных определяется спецификой решаемых задач. В рамках воркшопа рассматриваются две библиотеки, занимающие различные позиции на рынке инструментов визуализации.

**Chart.js** характеризуется следующими особенностями:

- низкий порог входа благодаря декларативному API и интуитивно понятной конфигурации;
- использование Canvas для рендеринга, что обеспечивает удовлетворительную производительность для малых и средних наборов данных;
- ограниченный набор встроенных типов графиков, расширяемый за счёт плагинов;
- наличие официальных обёрток для Vue.js, что упрощает интеграцию в компонентную архитектуру.

**ECharts (Apache)** отличается следующими характеристиками:

- широкий спектр встроенных типов графиков, включая тепловые карты, деревья, географические карты и свечные графики;
- высокая производительность при работе с большими объёмами данных благодаря оптимизированному Canvas-рендерингу;
- развитые интерактивные возможности: масштабирование, выделение областей, панель инструментов;
- более сложная конфигурация по сравнению с Chart.js, что обусловлено большим количеством опций.

### 2.2. Интеграция с Vue.js 3

При интеграции **Chart.js** в проект Vue.js 3 следует учитывать, что для работы с графиками необходимо зарегистрировать все используемые компоненты Chart.js, включая шкалы, типы элементов и плагины. Студенту следует обратить внимание на корректную передачу данных и опций конфигурации в компоненты-обёртки.

При работе с **ECharts** в среде Vue.js 3 критически важным аспектом является управление жизненным циклом компонентов. Для корректной работы необходимо:

- инициализировать экземпляр графика после монтирования компонента в DOM;
- обеспечивать уничтожение экземпляра графика при размонтировании компонента для предотвращения утечек памяти, особенно в приложениях с динамической сменой маршрутов;
- при изменении размеров контейнера или данных вызывать метод обновления графика.

### 2.3. Техника построения визуализаций сравнения

Для задач сравнения прогнозных значений с фактическими рекомендуется использовать следующие типы графиков:

| Тип графика | Применение |
|-------------|------------|
| Линейный график | Временные ряды, тренды, прогнозирование |
| Столбчатый график | Сравнение категорий, дискретные данные |
| Точечный график | Корреляционный анализ, выявление выбросов |
| Свечной график | Финансовые данные, анализ волатильности |

При визуализации результатов машинного обучения следует предусмотреть отображение метрик качества прогнозов: средней абсолютной ошибки (MAE), среднеквадратичной ошибки (RMSE), средней абсолютной процентной ошибки (MAPE).

## 3. Методические рекомендации по выполнению заданий

### 3.1. Задание 1. «Мой первый график» (разогрев)

Данное задание направлено на первичное знакомство с выбранной библиотекой и настройку рабочего окружения. Студентам необходимо загрузить данные из CSV-файла и построить столбчатую диаграмму, отображающую фактические и прогнозные значения продаж по месяцам.

При выполнении задания студенту следует обратить внимание на:

- корректную настройку импорта необходимых компонентов и плагинов библиотеки;
- использование реактивных переменных для хранения данных графика и их обновление после загрузки данных;
- правильную организацию процесса загрузки и парсинга CSV-данных;
- настройку легенды и цветовой схемы для различения серий данных.

### 3.2. Задание 2. Линейная регрессия + прогноз

Задание предполагает реализацию полного цикла: загрузка данных, обучение модели линейной регрессии, выполнение предсказаний на тестовой выборке и визуализация результатов.

Ключевые аспекты, на которые следует обратить внимание:

- корректное разделение данных на обучающую и тестовую выборки (80/20);
- правильная подготовка признакового пространства для обучения модели;
- расчёт и отображение метрик качества (MAE, RMSE);
- построение точечного графика с добавлением линии идеального прогноза для визуальной оценки качества модели.

### 3.3. Задание 3. Кластеризация клиентов

Задание направлено на визуализацию результатов кластеризации методом K-Means. Студентам необходимо реализовать алгоритм кластеризации, применить его к данным о доходах и расходах клиентов и отобразить результаты.

Особое внимание следует уделить:

- корректной реализации алгоритма K-Means с проверкой сходимости;
- визуальному разделению кластеров с использованием различных цветов;
- отображению центроидов специальным маркером, отличающимся от точек данных;
- реализации интерактивных подсказок, отображающих информацию о точке при наведении (доход, расходы, номер кластера).

### 3.4. Задание 4. Сравнение трёх моделей

Задание предполагает визуализацию прогнозов трёх моделей на одном графике: линейная регрессия, SVM и простое скользящее среднее.

При выполнении задания студенту следует обратить внимание на:

- единообразную подготовку данных для всех трёх моделей;
- визуальное различение моделей с использованием различных стилей линий (сплошная, пунктирная, штрихпунктирная) и цветов;
- отображение метрик качества для каждой модели для сравнительного анализа;
- организацию кода таким образом, чтобы добавление новой модели не требовало изменения существующей логики.

### 3.5. Задание 5. Финальный дашборд

Задание является итоговым и предполагает объединение всех предыдущих наработок в единый дашборд.

Ключевые аспекты реализации:

- **Реактивность:** все графики должны обновляться при изменении выбранной модели. Для этого следует использовать вычисляемые свойства или функцию отслеживания изменений.
- **Экспорт:** необходимо реализовать функцию сохранения графика в формате PNG.
- **Обновление данных:** требуется имитация реального времени через периодическое обновление данных, при этом частоту обновления следует ограничить для предотвращения избыточной нагрузки.
- **Компоновка:** использование CSS Grid или Flexbox для размещения трёх графиков и панели управления.

## 4. Критерии оценивания

Оценка выполнения воркшопа производится по совокупности критериев, установленных в материалах воркшопа. Максимальное количество баллов — 15.

| Критерий | Баллы |
|----------|-------|

| Работающий график (любой) | 2 |
| Интерактивность (tooltip, zoom) | 2 |
| ML модель (обучение + предсказание) | 4 |

| Финальный дашборд | 2 |
