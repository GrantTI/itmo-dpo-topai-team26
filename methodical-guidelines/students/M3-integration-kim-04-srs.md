# Методические рекомендации по выполнению самостоятельной работы
## Модуль 3. Самостоятельная работа «Развёртывание итогового приложения дисциплины в контейнерной среде»

---

## 1. Вводная часть

**Тема.** Перенос практик контейнерного развёртывания на репозиторий **собственного** проекта дисциплины.

**Цель.** Самостоятельно настроить и задокументировать воспроизводимый контур развёртывания; подтвердить доступность UI и метода API, связанного с моделью.

### Формат проведения

- **Тип работы:** самостоятельная работа
- **Формат:** индивидуальное выполнение
- **Сдача:** репозиторий с файлами развёртывания; демонстрация запуска; описание контура; рубежный контроль перед модулем 4

### Время выполнения

8 академических часов.

### Необходимые ресурсы

- Выполненные лабораторные работы «Развёртывание клиент-серверного приложения с моделями машинного обучения в контейнерной среде» и «Конфигурирование обратного прокси-сервера и многостадийная сборка контейнерных образов» (рабочие Dockerfile, Compose, Nginx) **либо** эквивалентный контур, собранный по методическим указаниям к этим работам
- Собственный проект с серверной и клиентской частями
- Docker, Docker Compose

---

## 2. Подготовка к выполнению

### Теоретический минимум

1. **Состав контура:** образы backend/frontend, Compose-сеть по именам сервисов, переменные окружения, Nginx как единая точка входа. Опора — практики лабораторных работ «Развёртывание клиент-серверного приложения с моделями машинного обучения в контейнерной среде» и «Конфигурирование обратного прокси-сервера и многостадийная сборка контейнерных образов», а также [1; 3].

2. **Воспроизводимость:** запуск по письменной инструкции без недокументированных ручных шагов (`cp .env.example .env` → `docker compose up --build -d` → проверка → `down`) [4; 5].

3. В этой работе оценивается **контур развёртывания**, а не качество обучения моделей, оформление UI, автотесты и презентация (это — проектная работа / модуль 4). Smoke API модели нужен как факт доступности контура с ML, а не как оценка качества модели [2].

4. **Граница с критерием 6 проекта:** полный балл за Compose/Nginx начисляется в этой самостоятельной работе «Развёртывание итогового приложения дисциплины в контейнерной среде». В проекте контур данной работы считается уже принятой базой; критерий 6 не требует повторной сдачи той же лабораторной «с нуля».

### Настройка рабочего окружения

1. Убедитесь, что проект запускается локально вне контейнеров.
2. Перенесите и адаптируйте файлы развёртывания с учебного стенда (лабораторные работы «Развёртывание клиент-серверного приложения с моделями машинного обучения в контейнерной среде», «Конфигурирование обратного прокси-сервера и многостадийная сборка контейнерных образов») под структуру своего репозитория.
3. Проверьте сценарий: чистая копия репозитория → следование инструкции → проверка UI и smoke API модели (не только health).

### Анализ задания

Инкремент относительно лабораторных работ «Развёртывание клиент-серверного приложения с моделями машинного обучения в контейнерной среде» и «Конфигурирование обратного прокси-сервера и многостадийная сборка контейнерных образов» — адаптация к **своему** проекту, инструкция и описание контура. Повторная сдача того же учебного стенда без адаптации **не засчитывается**.

## 3. Алгоритм выполнения

### Этап 1. Контур собственного проекта

Целевое дерево (имена каталогов допустимо адаптировать):

```text
my-project/
├── backend/
│   └── Dockerfile
├── frontend/
│   └── Dockerfile
├── nginx/
│   └── default.conf
├── docker-compose.yml
├── .env.example
├── README.md                # или DEPLOY.md — инструкция запуска
└── ...
```

1. Dockerfile для backend и frontend (допускается multi-stage, как в лабораторной работе «Конфигурирование обратного прокси-сервера и многостадийная сборка контейнерных образов»).
2. `docker-compose.yml` с сетью; взаимодействие по именам сервисов.
3. Параметры в переменных окружения; `.env.example` без секретов.
4. Nginx для публикации клиентского интерфейса и/или API.

Отдельный сервис инференса не обязателен: модель применяется внутри backend (модуль 1), если вы не выбрали иной паттерн для собственного проекта.

---

### Этап 2. Воспроизводимый запуск

1. Напишите инструкцию (раздел в README или `DEPLOY.md`): `cp .env.example .env`, `docker compose up --build -d`, адреса проверки, `docker compose down`.

**Шаблон фрагмента `DEPLOY.md`:**

```markdown
## Развёртывание

### Требования
- Docker 20.10+, Docker Compose 2.x

### Запуск
cp .env.example .env
docker compose up --build -d

### Проверка
- UI: http://localhost:8080
- API моделей (smoke): http://localhost:8080/api/models

### Останов
docker compose down

### Сервисы
| Сервис   | Назначение                          |
|----------|-------------------------------------|
| nginx    | Единая точка входа                  |
| frontend | Статика UI                          |
| backend  | FastAPI + in-process модель (M1)    |
```

2. Выполните запуск по инструкции; подтвердите UI и **метод API, связанный с моделью** (smoke; одного health недостаточно).
3. Останов и повторный запуск без ручного «лечения» контейнеров.
4. Проверьте отсутствие секретов в файлах и в истории Git.

```bash
# быстрая самопроверка перед сдачей
docker compose up --build -d
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8080/
curl -s http://localhost:8080/api/models | head
docker compose down
```
---

### Этап 3. Описание контура

В README / `DEPLOY.md` укажите:

1. Перечень сервисов Compose и назначение каждого.
2. Порты публикации и **имена** основных переменных окружения (без значений секретов).
3. Путь запроса: клиент → Nginx → backend → применение модели (внутри backend либо иной способ, зафиксированный в проекте).

## 4. Оформление результатов

**Сдать:** репозиторий с файлами из дерева §3; демонстрация запуска; описание контура.

Инструкция должна позволять другому обучающемуся повторить запуск без устных пояснений автора.

---

## 5. Критерии оценки

Максимум — **15 баллов**. Зачёт — не ниже **9**.

| Критерий | Баллы | Показатели |
|---|:---:|---|
| Полнота контура | 0–5 | Dockerfile, Compose, `.env.example`, Nginx для **своего** проекта |
| Воспроизводимый запуск | 0–5 | Запуск и останов; UI и smoke API модели; письменная инструкция |
| Описание контура | 0–3 | Сервисы, порты, путь запроса |
| Корректность конфигурации | 0–2 | Нет секретов в Git; ясная структура файлов |

**Не оцениваются здесь:** качество обучения моделей, оформление UI, покрытие тестами, презентация к защите проекта.

## 6. Самопроверка

- [ ] Контур относится к собственному проекту, а не только к стенду лабораторной работы «Развёртывание клиент-серверного приложения с моделями машинного обучения в контейнерной среде»
- [ ] Есть инструкция «с нуля»
- [ ] UI и smoke API модели доступны после запуска по инструкции
- [ ] Описаны сервисы и путь запроса
- [ ] Секретов в репозитории нет
- [ ] Понимаете: полный балл за контур — в этой самостоятельной работе «Развёртывание итогового приложения дисциплины в контейнерной среде», не повторная сдача в критерии 6 проекта
- [ ] Готовы повторить запуск на рубежном контроле
