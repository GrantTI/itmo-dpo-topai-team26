# Методические рекомендации по выполнению лабораторных работ по веб-технологиям

## 1. Общие положения

Настоящие методические рекомендации предназначены для студентов, приступающих к выполнению лабораторных работ по дисциплинам, связанным с веб-технологиями. Лабораторный практикум состоит из трёх последовательных работ, охватывающих базовые и продвинутые аспекты разработки веб-приложений: основы HTML и CSS, программирование на JavaScript с визуализацией данных, разработка интерфейса для выбора моделей машинного обучения с использованием фреймворка Vue.js 3 и бэкенда на FastAPI.

Выполнение работ предполагает освоение следующих компетенций:

- создание семантической разметки веб-страниц с использованием HTML5;
- применение каскадных таблиц стилей для визуального оформления элементов;
- программирование интерактивного поведения страниц на JavaScript;
- разработка компонентных приложений на фреймворке Vue.js 3;
- организация взаимодействия клиентской и серверной частей через REST API.

Продолжительность каждой лабораторной работы составляет 4–6 академических часов. Работы выполняются в строго установленной последовательности: HTML и CSS, затем JavaScript, затем Vue.js.

## 2. Лабораторная работа №1. Основы HTML и CSS

### 2.1. Цель и задачи

Целью работы является формирование базовых навыков создания веб-страниц с использованием языка гипертекстовой разметки HTML и каскадных таблиц стилей CSS. Студент должен освоить синтаксис основных элементов HTML, принципы стилизации и способы организации интерактивных элементов на странице.

Задачи работы:

1. Изучить структуру HTML-документа и иерархию заголовков.
2. Освоить создание списков различных типов, включая вложенные.
3. Научиться создавать гиперссылки с различными атрибутами и целевыми адресами.
4. Приобрести навыки встраивания изображений с заданными параметрами отображения.
5. Освоить проектирование веб-форм с различными типами полей ввода (текстовые поля, пароль, email, чекбоксы, радиокнопки, выпадающие списки, поле загрузки файлов).
6. Научиться применять CSS для базовой стилизации элементов и управления их состояниями (наведение, фокус, активное состояние).

### 2.2. Порядок выполнения

Работа включает десять последовательных заданий, каждое из которых расширяет функциональность создаваемой страницы. Студентам рекомендуется выполнять задания в приведённой последовательности, так как каждое последующее задание базируется на структуре, созданной в предыдущем.

**Задания 1–4** направлены на освоение базовых элементов HTML: заголовков, абзацев, списков, ссылок и изображений. При выполнении данных заданий следует обратить внимание на правильное использование семантических тегов и атрибутов, обеспечивающих доступность содержимого (атрибут `alt` для изображений, осмысленные тексты ссылок).

**Задания 5–8** посвящены работе с формами. Студенту необходимо создать форму регистрации, содержащую поля различных типов, обеспечив группировку связанных элементов через чекбоксы и радиокнопки с одинаковыми значениями атрибута `name`. Особое внимание следует уделить корректной привязке подписей к полям через атрибут `for` элемента `<label>`.

**Задания 9–10** посвящены каскадным таблицам стилей. На данном этапе студент применяет стили к ранее созданным элементам, используя селекторы по тегу, классу и псевдоклассы. Рекомендуется структурировать CSS-код, группируя правила по функциональному назначению: общие стили страницы, стили заголовков, стили форм, стили интерактивных состояний.

## 3. Лабораторная работа №2. Визуализация данных на JavaScript

### 3.1. Цель и задачи

Целью работы является освоение навыков разработки клиентской логики веб-приложений с использованием JavaScript. Студент должен научиться взаимодействовать с DOM-элементами, обрабатывать события, загружать и отображать данные, а также создавать визуализации с использованием элемента Canvas.

Задачи работы:

1. Освоить методы доступа к DOM-элементам и управления их стилями.
2. Изучить способы обработки событий (клик, изменение, фокус, потеря фокуса).
3. Научиться загружать данные из текстовых и CSV-файлов с использованием FileReader и Fetch API.
4. Освоить приёмы фильтрации и сортировки табличных данных.
5. Изучить основы визуализации данных с использованием элемента Canvas.
6. Сформировать навыки построения комплексных приложений, объединяющих загрузку, фильтрацию и визуализацию данных.

### 3.2. Порядок выполнения

Работа состоит из десяти заданий, сгруппированных по трём тематическим блокам.

**Задания 1–6 (базовые операции с DOM и событиями):**

- **Задание 1:** доступ к элементам и их стилизация через JavaScript.
- **Задание 2:** групповая обработка элементов по классам с использованием делегирования событий.
- **Задание 3:** обработка кликов по гиперссылкам с предотвращением перехода.
- **Задание 4:** обработка события изменения выпадающего списка (`onchange`).
- **Задание 5:** обработка событий получения и потери фокуса текстовыми полями с валидацией вводимых данных.
- **Задание 6:** реализация переключения видимости пароля с индикатором сложности.

В данных заданиях следует обратить внимание на корректное использование методов `addEventListener` и `removeEventListener`, понимание всплытия событий и возможность его остановки через `event.stopPropagation()`.

**Задания 7–8 (загрузка данных):**

- **Задание 7:** загрузка и отображение содержимого текстового файла с использованием FileReader, подсчёт строк, слов и символов.
- **Задание 8:** загрузка CSV-данных через Fetch API с сервера FastAPI, отображение в виде таблицы, реализация поиска и фильтрации.

При выполнении данных заданий следует освоить асинхронные операции (`async/await`), обработку ошибок запросов и работу с объектом Response.

**Задания 9–10 (визуализация):**

- **Задание 9:** создание столбчатой и линейной диаграмм на Canvas с подписями осей и интерактивной подсветкой при наведении.
- **Задание 10:** разработка комплексного приложения, включающего загрузку данных, табличное представление с фильтрацией и сортировкой, визуализацию выбранного типа графика и экспорт отфильтрованных данных в CSV.

При выполнении заданий 9–10 необходимо уделить внимание:

- масштабированию данных для корректного отображения на Canvas;
- обработке событий мыши для реализации интерактивности;
- оптимизации перерисовки при изменении параметров отображения.

## 4. Лабораторная работа №3. Разработка интерфейса для выбора ML-модели (Vue.js 3 + FastAPI)

### 4.1. Цель и задачи

Целью работы является освоение разработки полноценного веб-приложения с использованием фреймворка Vue.js 3 для клиентской части и FastAPI для серверной. Студент должен научиться создавать реактивные интерфейсы, динамические формы, взаимодействовать с бэкендом через REST API и визуализировать результаты работы алгоритмов машинного обучения.

Задачи работы:

1. Изучить архитектуру приложения «клиент-сервер» с разделением на фронтенд и бэкенд.
2. Освоить настройку проекта Vue.js 3 с использованием инструмента сборки Vite.
3. Научиться создавать реактивные компоненты с использованием Composition API и директив `v-model`, `v-for`, `v-if`.
4. Освоить механизмы прокси-сервера Vite для перенаправления запросов к бэкенду.
5. Изучить реализацию динамических форм на основе метаданных, получаемых от сервера.
6. Научиться выполнять HTTP-запросы с использованием библиотеки Axios.
7. Освоить обработку и визуализацию результатов работы моделей машинного обучения.
8. Изучить реализацию серверной части на FastAPI с поддержкой CORS.

### 4.2. Порядок выполнения

Работа выполняется в четыре этапа, каждый из которых завершается проверкой работоспособности соответствующей части приложения.

**Этап 1: Настройка проекта (30 минут)**

1. Создать структуру директорий проекта: `backend/` и `frontend/` с соответствующими поддиректориями.
2. Настроить бэкенд FastAPI: создать файл `requirements.txt` с зависимостями, реализовать базовое приложение с настройкой CORS, описать Pydantic-модели `ModelConfig` и `ModelMetrics`, определить структуру `AVAILABLE_MODELS` с метаданными доступных моделей машинного обучения.
3. Настроить фронтенд: создать `package.json` с зависимостями (Vue 3, Axios, Vite), настроить `vite.config.js` с прокси-сервером для перенаправления запросов с `/api` на `http://localhost:8000`.

**Этап 2: Разработка Vue.js компонентов (2 часа)**

1. Создать точку входа `main.js` и корневой компонент `App.vue`.
2. Реализовать API-клиент (`frontend/src/api/index.js`) с методами:
   - `getModels()` — получение списка доступных моделей;
   - `trainModel(config)` — отправка параметров модели на обучение;
   - `uploadData(file)` — загрузка CSV-файла с данными.
3. В корневом компоненте реализовать:
   - загрузку списка моделей в хуке `onMounted()`;
   - отображение выпадающего списка моделей с использованием `v-for`;
   - динамическую генерацию полей ввода на основе параметров выбранной модели (для параметров типа `boolean` — чекбокс, `number` — числовое поле, `select` — выпадающий список);
   - загрузку CSV-файла через элемент `<input type="file">` с отправкой на сервер через `FormData`;
   - отправку конфигурации модели на обучение и отображение полученных метрик.

**Этап 3: Реализация бэкенда с ML-моделями (1.5 часа)**

1. Создать класс `MLService` с методами:
   - `load_data(file_path)` — загрузка данных из CSV, разделение на признаки и целевую переменную, преобразование категориальных переменных через `pd.get_dummies()`, масштабирование признаков, разделение на обучающую и тестовую выборки;
   - `train_random_forest(params)` — обучение модели Random Forest с вычислением метрик;
   - `train_linear_regression(params)` — обучение линейной регрессии;
   - `train_svm(params)` — обучение SVM.
2. Интегрировать `MLService` в эндпоинты FastAPI:
   - `POST /api/upload-data` — загрузка файла, создание экземпляра сервиса, вызов `load_data()`;
   - `POST /api/train` — выбор метода обучения по `model_id`, вызов соответствующего метода с переданными параметрами, возврат метрик.
3. Реализовать обработку ошибок: проверка наличия загруженных данных, существования модели, обработка исключений при обучении.

**Этап 4: Запуск и тестирование (30 минут)**

1. Запустить бэкенд-сервер: `uvicorn app.main:app --reload`.
2. Запустить фронтенд-сервер: `npm run dev`.
3. Выполнить тестирование приложения с использованием предоставленного файла `test_data.csv`:
   - загрузить данные через интерфейс;
   - выбрать модель и настроить гиперпараметры;
   - выполнить обучение и проверить корректность отображаемых метрик.
4. Проверить обработку ошибок при некорректных действиях пользователя.
