# Методические указания для студентов по подготовке к тестированию по веб-технологиям (HTML, CSS, JavaScript, Vue.js)

## 1. Введение

Настоящие методические указания предназначены для организации самостоятельной подготовки студентов к тестированию по дисциплинам, связанным с веб-технологиями. Тестовые задания охватывают четыре основных раздела: язык гипертекстовой разметки HTML, каскадные таблицы стилей CSS, язык программирования JavaScript и фреймворк Vue.js 3.

Первые два раздела (HTML и CSS) объединены в один тест, третий раздел посвящён JavaScript, четвёртый — Vue.js. Количество вопросов и шкалы оценивания различаются в зависимости от раздела: тесты по HTML/CSS и JavaScript содержат по 30 вопросов, тест по Vue.js — 20 вопросов. Каждый вопрос предполагает выбор одного или нескольких правильных вариантов из предложенных. Успешное выполнение теста требует систематического повторения теоретического материала и отработки практических навыков.

## 2. Структура и критерии оценивания

Результаты тестирования по разделам HTML/CSS и JavaScript оцениваются по четырёхуровневой шкале в зависимости от числа правильных ответов (максимальный балл — 3).

| Балл | Количество правильных ответов (из 30) | Процент выполнения |
|------|---------------------------------------|-------------------|
| 3 | 25 – 30 | 83 – 100 % |
| 2 | 18 – 24 | 60 – 82 % |
| 1 | 10 – 17 | 33 – 59 % |
| 0 | 0 – 9 | 0 – 32 % |

Для теста по Vue.js 3 применяется пятиуровневая шкала (максимальный балл — 4) с учётом 20 вопросов.

| Балл | Количество правильных ответов (из 20) | Процент выполнения |
|------|---------------------------------------|-------------------|
| 4 | 18 – 20 | 90 – 100 % |
| 3 | 14 – 17 | 70 – 89 % |
| 2 | 10 – 13 | 50 – 69 % |
| 1 | 5 – 9 | 25 – 49 % |
| 0 | 0 – 4 | 0 – 24 % |

## 3. Содержание разделов

### 3.1. HTML

В данном разделе проверяется знание синтаксиса и семантики элементов HTML. Обязательному усвоению подлежат следующие темы:

- **Структурные элементы документа:** `<html>`, `<head>`, `<body>`.
- **Заголовки:** `<h1>` – `<h6>` (от наиболее значимого к наименее значимому).
- **Текстовые блоки и списки:** абзацы (`<p>`), маркированные (`<ul>`) и нумерованные (`<ol>`) списки, элементы списка (`<li>`).
- **Таблицы:** контейнер `<table>`, строки `<tr>`, ячейки `<td>` и `<th>`.
- **Гиперссылки и изображения:** элемент `<a>` с атрибутом `href`, элемент `<img>` с атрибутом `src`.
- **Формы:** контейнер `<form>`, типы полей ввода `<input>` (`text`, `password`, `checkbox`, `radio`, `submit`), многострочное текстовое поле `<textarea>`, выпадающий список `<select>` с вложенными `<option>`. Атрибуты: `for` элемента `<label>` для связывания с идентификатором поля, `required` для указания обязательности заполнения, `multiple` для поддержки множественного выбора в `<select>`. Различие между `checkbox` (допускает множественный выбор) и `radio` (в группе с одинаковым `name` допускает выбор только одного варианта).
- **Мультимедиа:** элемент `<video>` для встраивания видео.

### 3.2. CSS

Раздел посвящён правилам применения стилей к элементам HTML. Необходимо знать:

- **Типы селекторов:**
  - селектор по имени тега (например, `p`, `div`);
  - селектор по классу (`.className`);
  - селектор по идентификатору (`#idName`);
  - универсальный селектор (`*`);
  - комбинированные селекторы: вложенный (`div p` — все `p` внутри `div`), дочерний (`div > p` — только прямые потомки), селектор по тегу с классом (`p.example`), селектор по нескольким классам (`.class1.class2` — элемент, содержащий оба класса).
- **Специфичность селекторов:** убывание приоритета: идентификатор → класс → тег.
- **Основные стилевые свойства:**
  - шрифтовое оформление: `font-family`, `font-size`, `font-weight`;
  - блочная модель: `margin` (внешний отступ), `padding` (внутренний отступ), `border` (граница);
  - позиционирование: значения `static`, `relative`, `absolute` (позиционирование относительно ближайшего позиционированного предка), `fixed` (фиксация относительно окна просмотра);
  - режим отображения: `inline`, `block`, `inline-block`.

### 3.3. JavaScript

Данный раздел охватывает основы языка и взаимодействие с объектной моделью документа (DOM).

- **Методы поиска элементов:**
  - `document.getElementById('id')` — возвращает элемент или `null`;
  - `document.getElementsByTagName('p')` — возвращает живую коллекцию `HTMLCollection`;
  - `document.getElementsByClassName('class')` — возвращает живую коллекцию (при отсутствии элементов — пустую коллекцию с длиной 0);
  - `document.querySelector('.class')` — возвращает первый соответствующий элемент;
  - `document.querySelectorAll('.class')` — возвращает статический `NodeList`.
  - **Различие:** методы `getElementsBy*` возвращают «живые» коллекции, автоматически обновляемые при изменении DOM; `querySelectorAll` возвращает статическую коллекцию.
- **Свойства элементов:** `innerHTML` (содержимое с разметкой), `textContent` (текстовое содержимое без тегов), `value` (значение полей ввода), `className` и `classList` (для управления классами).
- **Обработка событий:**
  - событие клика — `onclick`;
  - получение и потеря фокуса — `onfocus` и `onblur` соответственно;
  - изменение значения поля: `oninput` (срабатывает при каждом изменении), `onchange` (срабатывает после подтверждения изменения);
  - остановка всплытия события — метод `event.stopPropagation()`.
- **Инструменты отладки:** вывод в консоль — `console.log()`, табличное представление — `console.table()`, вывод ошибок — `console.error()`. Установка точек останова осуществляется через директиву `debugger;` или в панели «Sources» инструментов разработчика.
- **Типы данных и сравнение:**
  - Результат операции `typeof null` равен `"object"` (особенность языка).
  - Значение `NaN` (Not a Number) используется для обозначения нечисловых результатов.
  - Нестрогое сравнение (`==`) выполняет приведение типов: `5 == "5"` → `true`. Строгое сравнение (`===`) не выполняет приведение: `5 === "5"` → `false`.
  - Ложные (`falsy`) значения в булевом контексте: `null`, `undefined`, `0`, пустая строка, `NaN`, `false`. Все непустые строки, включая `"false"`, являются истинными (`truthy`).

### 3.4. Vue.js 3

Раздел посвящён практическому применению фреймворка Vue.js версии 3, включая инструменты сборки, композиционный API, взаимодействие с серверной частью и интеграцию сторонних библиотек. Студент должен владеть следующими темами:

- **Создание проекта:** использование команды `npm create vite@latest` с шаблоном `vue`; структура каталогов и конфигурационные файлы (`vite.config.js`).
- **Настройка прокси-сервера Vite:** конфигурация `server.proxy` в `vite.config.js` для перенаправления запросов к бэкенд-серверу (например, FastAPI) с целью обхода политики CORS.
- **Хуки жизненного цикла (Composition API):** применение `onMounted()` для выполнения асинхронных операций (например, HTTP-запросов) после монтирования компонента в DOM; отличие от `created()` и `beforeMount()`.
- **Обработка форм:** использование директивы `v-model` для двусторонней привязки данных полей ввода; применение модификатора `.prevent` к событию `@submit` для отмены стандартной отправки формы; отправка данных через `fetch` или `axios`.
- **HTTP-клиенты:** установка пакета `axios` через `npm`; выполнение POST-запросов с передачей JSON-данных; обработка ответов и ошибок с использованием `try/catch` или метода `.catch()`.
- **Интеграция библиотек визуализации:** установка и использование Chart.js (через `npm install chart.js` и импорт `chart.js/auto`), ECharts (прямой импорт или через обёртку `vue-echarts`); инициализация графиков в хуке `onMounted`.
- **Работа с файлами:** отправка файлов на сервер с использованием объекта `FormData` и заголовка `Content-Type: multipart/form-data`; получение ссылки на элемент `<input type="file">` через референсы (`ref`).
- **Маршрутизация:** установка `vue-router@4` для Vue.js 3; настройка маршрутов.
- **Препроцессоры CSS:** установка пакета `sass` и использование атрибута `lang="scss"` в теге `<style>` для поддержки SCSS-синтаксиса.
- **Отладка:** использование расширения Vue Devtools для анализа состояния компонентов, событий и производительности; применение `console.log()` и стандартных инструментов разработчика браузера.
- **Взаимодействие с сервером (FastAPI):** настройка CORS на стороне сервера с использованием `CORSMiddleware`; понимание того, что прокси-сервер Vite перенаправляет запросы с порта разработки (например, 3000) на порт бэкенда (например, 8000), при этом запрос в коде отправляется на относительный путь `/api/...`.

## 4. Методические рекомендации по подготовке

### 4.1. Теоретическая подготовка

Рекомендуется систематизировать знания по каждому разделу:

- **Для HTML/CSS:** составить краткий конспект основных элементов, атрибутов, селекторов и свойств, уделяя особое внимание формам, спискам и блочной модели.
- **Для JavaScript:** отдельно проработать правила приведения типов, различия между строгим и нестрогим сравнением, а также особенности возвращаемых коллекций методами DOM.
- **Для Vue.js:** изучить документацию по Composition API, хукам жизненного цикла, директивам (`v-model`, `v-for`, `v-if`), настройке прокси и интеграции с внешними библиотеками. Обратить внимание на различия между Options API и Composition API, хотя в тесте акцент сделан на Composition.

### 4.2. Практическая подготовка

Целесообразно выполнить следующие действия:

1. Решить демонстрационные варианты тестов (по 30 вопросов для HTML/CSS и JS, 20 для Vue) без использования справочных материалов, с последующей самопроверкой по приведённым ответам.
2. При обнаружении ошибок провести анализ неправильных ответов, обращаясь к официальной документации для уточнения соответствующих правил.
3. Для разделов JavaScript и Vue.js использовать среду разработчика в браузере и инструменты командной строки для экспериментальной проверки синтаксиса и поведения конструкций, вызывающих сомнения (например, выполнение операций сравнения в консоли, настройка прокси в тестовом проекте).
4. Для Vue.js рекомендуется создать небольшой проект с использованием Vite, настроить прокси к тестовому бэкенду (например, FastAPI), реализовать форму с отправкой данных, подключить библиотеку графиков и отработать обработку ошибок.

### 4.3. Стратегия выполнения теста

В процессе тестирования рекомендуется:

- Внимательно читать формулировку каждого вопроса, обращая особое внимание на ключевые слова (например, «неверно», «отличается», «все», «кроме»), выделенные в тексте.
- Применять метод исключения заведомо некорректных вариантов для повышения вероятности правильного выбора.
- При затруднении с ответом на конкретный вопрос переходить к следующим, возвращаясь к пропущенным после завершения остальных заданий.
- В вопросах по JavaScript, содержащих операторы сравнения (`==` или `===`), учитывать правила приведения типов.
- В вопросах по Vue.js обращать внимание на синтаксис команд, конфигурационных файлов и названия хуков, так как в тесте часто проверяется точное знание имён и параметров.

## 5. Рекомендуемые источники для подготовки

- **MDN Web Docs (русскоязычная версия)** — наиболее полный и авторитетный справочник по HTML, CSS и JavaScript. Используется для уточнения деталей и проверки спорных вопросов.  
  URL: https://developer.mozilla.org/ru

- **Современный учебник JavaScript (learn.javascript.ru)** — систематическое изложение основ языка, включая DOM, события, методы поиска элементов и типы данных. Материал полностью покрывает тематику раздела JavaScript.  
  URL: https://learn.javascript.ru

- **Официальная документация Vue.js** — содержит исчерпывающую информацию по всем аспектам фреймворка, включая руководство по Composition API, реактивности, маршрутизации и интеграции с инструментами сборки. Рекомендуется для подготовки по разделу Vue.js.  
  URL: https://vuejs.org (доступна русскоязычная версия).

Дополнительно для практической отработки навыков работы с Vue.js и сопутствующими библиотеками (`axios`, Chart.js, Vite) полезно обращаться к официальным репозиториям и руководствам этих инструментов. При подготовке рекомендуется начинать с изучения документации Vue.js, затем переходить к практическим упражнениям по созданию проекта и взаимодействию с API, а при возникновении неоднозначных ситуаций сверяться с MDN и учебником по JavaScript.
