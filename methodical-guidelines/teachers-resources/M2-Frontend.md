# Методические рекомендации для преподавателей по использованию ресурсов при разработке контрольно-измерительных материалов

## 1. Общие положения

Настоящие рекомендации предназначены для преподавателей, осуществляющих разработку и применение контрольно-измерительных материалов (КИМ) по модулю «Frontend» в рамках дисциплины «Технологии развертывания моделей машинного обучения в клиент-серверных приложениях». Рекомендации охватывают вопросы подбора ресурсов для составления тестов, лабораторных работ и воркшопа, а также принципы их применения в учебном процессе.

КИМ разработаны на основе трёх тестов («Тест по HTML и CSS», «Тест по JavaScript», «Тест по Vue.js 3»), трёх лабораторных работ («Визуализация данных в ML моделях. Работа с HTML и CSS», «Визуализация данных в ML моделях. JavaScript», «Разработка интерфейса для выбора ML модели») и воркшопа («Визуализация графиков для ML моделей: ECharts vs Chart.js»).

## 2. Подбор ресурсов для разработки тестовых заданий

### 2.1. Принципы отбора содержания

При разработке тестовых заданий целесообразно руководствоваться следующими принципами:

- **Соответствие рабочей программе:** отбираются темы, зафиксированные в рабочей программе дисциплины. Для «Теста по HTML и CSS» — семантическая разметка, формы, селекторы, свойства CSS. Для «Теста по JavaScript» — DOM-методы, события, типы данных, операторы сравнения. Для «Теста по Vue.js 3» — создание проектов, настройка прокси, хуки жизненного цикла, директивы, интеграция с API.

- **Практическая значимость:** задания ориентированы на проверку знаний, непосредственно применяемых в лабораторных работах и воркшопе. Например, вопросы о настройке CORS и прокси в тесте по Vue.js 3 непосредственно связаны с лабораторной работой «Разработка интерфейса для выбора ML модели».

- **Разграничение уровней сложности:** тесты включают вопросы базового уровня (знание синтаксиса), среднего уровня (понимание взаимосвязей) и продвинутого уровня (анализ неочевидных ситуаций, например, `typeof null` в JavaScript).

### 2.2. Ресурсная база для составления заданий

- **Официальная документация:** Основой для формулировки корректных ответов служат спецификации HTML5, CSS, ECMAScript и документация Vue.js. При возникновении спорных ситуаций приоритет имеет официальная документация.

- **Примеры из лабораторных работ:** Задания тестов целесообразно связывать с конкретными заданиями лабораторных работ. Например, вопросы о типах полей ввода в «Тесте по HTML и CSS» соотносятся с заданиями 5–8 лабораторной работы «Визуализация данных в ML моделях. Работа с HTML и CSS».

- **Типичные ошибки студентов:** Анализ ошибок, допускаемых студентами при выполнении лабораторных работ, позволяет сформулировать вопросы, проверяющие понимание сложных тем (различие между «живыми» и статическими коллекциями, строгое и нестрогое сравнение).

### 2.3. Форматы тестовых заданий

Для тестов используются вопросы с множественным выбором ответа (один или несколько правильных вариантов). Рекомендуется следующее распределение:

- **Базовый уровень (40% вопросов):** прямое воспроизведение знаний (например, «Какой тег используется для создания маркированного списка?»).
- **Средний уровень (40% вопросов):** применение знаний в типовой ситуации (например, «Какой селектор выберет все элементы `<p>` внутри `<div>`?»).
- **Продвинутый уровень (20% вопросов):** анализ нестандартных ситуаций (например, «Что вернет `Boolean("false")`?»).

## 3. Подбор ресурсов для разработки лабораторных работ

### 3.1. Структурирование по уровням сложности

Лабораторные работы построены по принципу «от простого к сложному». Первые задания каждой работы направлены на отработку базовых навыков, последующие — на решение комплексных задач.

- **Лабораторная работа «Визуализация данных в ML моделях. Работа с HTML и CSS»:** Задания 1–4 (заголовки, списки, ссылки, изображения) — базовый уровень; задания 5–8 (формы, чекбоксы, выпадающие списки, загрузка файлов) — средний уровень; задания 9–10 (стилизация, интерактивные состояния) — продвинутый уровень.

- **Лабораторная работа «Визуализация данных в ML моделях. JavaScript»:** Задания 1–6 (DOM, события, валидация) — базовый и средний уровни; задания 7–9 (загрузка файлов, работа с CSV, Canvas) — средний и продвинутый уровни; задание 10 (комплексное приложение) — интегративный уровень.

- **Лабораторная работа «Разработка интерфейса для выбора ML модели»:** Этап 1 (настройка проекта) — базовый; этап 2 (компоненты Vue) — средний; этап 3 (бэкенд с ML) — продвинутый.

### 3.2. Инструментарий для выполнения

Для выполнения лабораторных работ студентам необходим следующий набор ресурсов:

- **Редактор кода:** Visual Studio Code с расширениями Volar (для Vue.js), ESLint, Prettier.
- **Среда выполнения:** Node.js 18+, npm, Python 3.9+.
- **Браузер:** Google Chrome или Firefox с расширением Vue Devtools.
- **Данные:** файлы `test_data.csv` для лабораторной работы по Vue.js, `sample.txt` и `sample.csv` для лабораторной работы по JavaScript.

Преподавателю рекомендуется предоставить студентам шаблоны проектов для лабораторной работы по Vue.js (структура директорий, файлы `package.json` и `vite.config.js`).

## 4. Подбор ресурсов для разработки воркшопа

### 4.1. Выбор библиотек для сравнительного анализа

Воркшоп **«Визуализация графиков для ML моделей: ECharts vs Chart.js»** предполагает командную работу с двумя библиотеками. При выборе библиотек учитывались следующие критерии:

- **Распространённость:** обе библиотеки широко используются в индустрии.
- **Различия в подходе:** Chart.js — декларативный, с низким порогом входа; ECharts — конфигурационный, с широкими возможностями.
- **Применимость к задачам ML:** обе библиотеки поддерживают точечные, линейные и столбчатые графики, необходимые для визуализации результатов моделей.

### 4.2. Данные для воркшопа

Для воркшопа подготовлены синтетические наборы данных:

- `sales_forecast.csv` — для визуализации временных рядов.
- `housing.json` — для задач регрессии.
- `customers.csv` — для кластеризации.

Преподавателю рекомендуется заранее раздать файлы студентам или разместить их в репозитории проекта.

### 4.3. Методические ресурсы для команд

Каждая команда получает:

- Инструкцию по установке и настройке библиотеки.
- Шаблон composable-функции для загрузки данных.
- Примеры реализации ML-моделей (линейная регрессия, K-Means, SVM).

## 5. Рекомендации по применению ресурсов в учебном процессе

### 5.1. Интеграция тестов, лабораторных работ и воркшопа

Рекомендуется следующая последовательность использования КИМ:

1. **Тест по HTML и CSS** — после изучения основ разметки и стилей, перед выполнением лабораторной работы «Визуализация данных в ML моделях. Работа с HTML и CSS».
2. **Лабораторная работа «Визуализация данных в ML моделях. Работа с HTML и CSS»** — практическое закрепление навыков.
3. **Тест по JavaScript** — после изучения основ языка, перед выполнением лабораторной работы «Визуализация данных в ML моделях. JavaScript».
4. **Лабораторная работа «Визуализация данных в ML моделях. JavaScript»** — отработка навыков работы с DOM, событиями и визуализацией.
5. **Тест по Vue.js 3** — после изучения фреймворка, перед лабораторной работой «Разработка интерфейса для выбора ML модели».
6. **Лабораторная работа «Разработка интерфейса для выбора ML модели»** — комплексное применение навыков.
7. **Воркшоп «Визуализация графиков для ML моделей: ECharts vs Chart.js»** — завершающий этап, интегрирующий все изученные технологии.

### 5.2. Адаптация ресурсов под уровень студентов

При работе со студентами разных уровней подготовки преподаватель может:

- Для базового уровня предлагать готовые шаблоны кода и пошаговые инструкции.
- Для продвинутого уровня предоставлять только постановку задачи без шаблонов.
- Использовать дополнительные задания (например, добавление новых типов графиков в воркшопе или новых ML-моделей в лабораторной работе).

### 5.3. Контроль использования ресурсов

Для предотвращения недобросовестного использования ресурсов рекомендуется:

- Проводить защиту лабораторных работ с демонстрацией кода и объяснением ключевых решений.
- В тестах использовать вопросы с множественным выбором, требующие понимания, а не механического запоминания.
- В воркшопе оценивать командную презентацию и способность аргументировать выбор библиотеки.
