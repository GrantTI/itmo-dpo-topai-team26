# Детализация критериев оценивания по модулю проект.

### Критерии оценивания
Детализированная таблица критериев и показателей оценивания выполнения проектного задания для преподавателей

| № | Критерий | Проверяемые элементы | Детализация оценивания | Баллы |
|---|---|---|---|---:|
| 1 | Постановка задачи и обоснование решения | Цель проекта сформулирована корректно, выбранный стек технологий и архитектурный подход обоснованы | **10–9 баллов:** цель проекта и тип задачи сформулированы чётко, выбор модели и стека технологий обоснован с учётом данных, требований и сценария применения.<br>**8–6 баллов:** цель и задача определены верно, но обоснование выбора модели или технологий неполное.<br>**5–3 балла:** постановка задачи поверхностная, выбор модели и технологий объяснён слабо.<br>**2–1 балл:** цель и модель указаны формально, связь между задачей и выбранным решением не прослеживается.<br>**0 баллов:** постановка задачи отсутствует или не соответствует проекту. | 0–10 |
| 2 | Архитектура и организация приложения | Реализована корректная клиент-серверная архитектура, логика приложения структурирована, учтены связи компонентов | **10–9 баллов:** архитектура описана и реализована корректно, компоненты разделены, взаимодействие между ними прозрачно.<br>**8–6 баллов:** архитектура в целом верна, но есть отдельные неточности в организации компонентов или потоков данных.<br>**5–3 балла:** структура приложения частично нарушена, разделение ролей компонентов недостаточно выражено.<br>**2–1 балл:** архитектура слабо продумана, связи между компонентами неясны.<br>**0 баллов:** архитектура отсутствует или не соответствует заданию. | 0–10 |
| 3 | Реализация backend на Python | Код приложения корректен, работоспособен, соответствует поставленной задаче, используются возможности Python и прикладные библиотеки | **10–9 баллов:** backend работает стабильно, код структурирован, используются подходящие библиотеки Python, логика реализована полно.<br>**8–6 баллов:** backend в целом работоспособен, но есть отдельные недочёты в структуре кода или реализации функций.<br>**5–3 балла:** backend частично работает, отдельные функции реализованы неполно или нестабильно.<br>**2–1 балл:** реализация backend фрагментарна, существенные ошибки мешают использованию.<br>**0 баллов:** backend не реализован. | 0–10 |
| 4 | Реализация API для работы с данными | Обеспечены загрузка, проверка, очистка, предпросмотр и подготовка данных, корректно реализованы сценарии работы с CSV и целевой переменной | **20–17 баллов:** данные загружаются и подготавливаются корректно, учтены пропуски, типы данных, кодирование, нормализация/масштабирование при необходимости, подготовка соответствует модели.<br>**16–13 баллов:** основные этапы подготовки данных выполнены, но часть преобразований реализована не полностью или без достаточного обоснования.<br>**12–8 баллов:** реализованы только отдельные этапы подготовки, есть ошибки в обработке данных или признаков.<br>**7–1 балл:** подготовка данных формальна, качество входных данных не обеспечено.<br>**0 баллов:** подготовка данных отсутствует. | 0–20 |
| 5 | Развертывание и использование модели МО | Модель интегрирована в приложение, обеспечены вызов, применение и сопровождение модели через API | **20–17 баллов:** в приложение интегрирована обученная модель, она корректно загружается и используется через API без сбоев.<br>**16–13 баллов:** модель работает, но есть недочёты в сохранении, загрузке или интеграции с API.<br>**12–8 баллов:** модель используется нестабильно либо интеграция реализована частично.<br>**7–1 балл:** модель формально присутствует, но практическое применение в приложении не обеспечено.<br>**0 баллов:** модель не подключена к проекту. | 0–20 |
| 6 | Промышленные практики разработки и эксплуатации | Применены инструменты развертывания, обработки ошибок, документирования и сопровождения | **10–9 баллов:** предусмотрены документация, обработка ошибок, инструкции по запуску, решение пригодно для эксплуатации.<br>**8–6 баллов:** основные элементы сопровождения есть, но оформлены не полностью или недостаточно детально.<br>**5–3 балла:** документация и эксплуатационные элементы представлены частично.<br>**2–1 балл:** практики развертывания и сопровождения отражены формально.<br>**0 баллов:** отсутствуют эксплуатационные и сопровождательные элементы. | 0–10 |
| 7 | Качество кода и тестируемость | Структура проекта логична, код читаем, предусмотрены тесты или проверочные сценарии (выполняются успешно), соблюдены базовые соглашения оформления | **10–9 баллов:** код аккуратный, логичный, есть тесты или сценарии проверки, структура проекта понятна.<br>**8–6 баллов:** код в целом хороший, но отдельные части нуждаются в доработке или упрощении.<br>**5–3 балла:** читаемость и тестируемость ограничены, структура местами перегружена или неясна.<br>**2–1 балл:** код слабо структурирован, проверить работу решения затруднительно.<br>**0 баллов:** качество кода не позволяет оценить работоспособность проекта. | 0–10 |
| 8 | Защита проекта | Студент уверенно представляет решение, причины выбора решения, объясняет архитектуру, инструменты, ограничения и результаты | **10–9 баллов:** студент уверенно отвечает на вопросы, объясняет решения и демонстрирует понимание проекта.<br>**8–6 баллов:** ответы в целом верные, но часть пояснений неполная или неточная.<br>**5–3 балла:** студент затрудняется в объяснении архитектуры, подготовки данных или выбора модели.<br>**2–1 балл:** защита формальная, понимание проекта проявлено слабо.<br>**0 баллов:** студент не смог защитить проект. | 0–10 |
|  | **Итого** |  |  | **100** |

### Итоговая характеристика
| Набранные баллы | Описание |
|---|---|
| 86–100 | Обучающийся демонстрирует полное и уверенное владение материалом, успешно выполняет все этапы проектного задания, обосновывает принятые решения и корректно защищает проект. |
| 71–85 | Обучающийся в целом правильно выполняет проектное задание, допускает отдельные неточности, но показывает достаточный уровень владения материалом и способен объяснить основные решения. |
| 56–70 | Обучающийся выполняет основные требования задания, но демонстрирует неполное понимание отдельных этапов, допускает существенные недочёты в реализации или защите проекта. |
| 0–55 | Обучающийся не выполняет значительную часть требований задания, проект имеет существенные ошибки или не может быть в полной мере защищён. |
