# Методические рекомендации по преподаванию модуля «Frontend» в дисциплине «Технологии развертывания моделей машинного обучения в клиент-серверных приложениях»

## 1. Общие положения

Настоящие методические рекомендации предназначены для преподавателя, реализующего модуль «Frontend» в рамках дисциплины. Модуль включает три вида учебных мероприятий: тестирование (по HTML/CSS, JavaScript и Vue.js 3), лабораторные работы (три последовательные работы) и воркшоп по сравнительному анализу библиотек визуализации. Рекомендации содержат информацию об организации учебного процесса, необходимом программном обеспечении, формах представления результатов и типичных трудностях студентов.

## 2. Программное обеспечение для студентов

Для выполнения всех видов работ студенту необходим следующий набор программных средств:

- **Редакторы кода:** Рекомендуется использование Visual Studio Code с расширениями для Vue.js (Volar), ESLint и Prettier. Допускается использование альтернативных редакторов (Sublime Text, WebStorm).
- **Браузеры:** Для разработки и отладки рекомендуется Google Chrome или Mozilla Firefox с установленным расширением Vue Devtools.
- **Среда выполнения:** Требуется Node.js версии 18 или выше для работы с пакетным менеджером npm и инструментом сборки Vite.
- **Интерпретатор Python:** Для запуска серверной части FastAPI требуется Python версии 3.9 или выше.
- **Дополнительные инструменты:** Для работы с форматом CSV в лабораторных работах может потребоваться библиотека papaparse (устанавливается через npm).

Преподавателю рекомендуется предоставить студентам файл с перечнем программного обеспечения на первом занятии, а также проверить наличие установленных инструментов перед началом выполнения лабораторных работ.

## 3. Тестирование: структура и проведение

Тестирование проводится в письменной или электронной форме и включает три теста, соответствующих последовательности изучения материала. Тесты рекомендуется проводить после завершения теоретической части каждого раздела.

- **Тест по HTML и CSS** включает 30 вопросов, охватывающих семантическую разметку, формы, селекторы, свойства и специфичность CSS. Время выполнения — 30–40 минут.
- **Тест по JavaScript** включает 30 вопросов, посвящённых методам DOM, обработке событий, типам данных и операторам сравнения. Время выполнения — 30–40 минут.
- **Тест по Vue.js 3** включает 20 вопросов по созданию проектов, настройке прокси, хукам жизненного цикла, директивам и интеграции с API. Время выполнения — 20–30 минут.

**Критерии оценивания:**
- Для HTML/CSS и JavaScript: максимальный балл — 3 (25–30 правильных ответов).
- Для Vue.js: максимальный балл — 4 (18–20 правильных ответов).

Преподавателю рекомендуется предоставить студентам демонстрационные варианты тестов за 1–2 недели до проведения для ознакомления с форматом вопросов. После проведения теста целесообразно организовать разбор типичных ошибок на следующем занятии.

## 4. Лабораторные работы: организация выполнения

Лабораторные работы выполняются последовательно: сначала HTML и CSS, затем JavaScript, затем Vue.js 3 и FastAPI. Каждая работа включает 10 заданий возрастающей сложности. Продолжительность каждой работы составляет 4–6 академических часов.

### 4.1. Лабораторная работа №1 (HTML и CSS)

Студент создаёт статическую веб-страницу с заголовками, списками, гиперссылками, изображениями и формами, после чего применяет CSS-стилизацию. Преподавателю рекомендуется контролировать использование семантических тегов и корректность связывания подписей с полями форм через атрибут `for`. Выполнение данной работы не требует установки дополнительного программного обеспечения кроме редактора кода и браузера.

### 4.2. Лабораторная работа №2 (JavaScript)

Студент разрабатывает интерактивные элементы (переключение видимости пароля, валидация форм, загрузка файлов), реализует загрузку данных из CSV через FastAPI и строит диаграммы на Canvas. Для выполнения заданий 8–10 требуется запуск FastAPI-сервера, предоставленного в материалах. Преподавателю следует обратить внимание на освоение студентами асинхронных операций и обработки ошибок.

### 4.3. Лабораторная работа №3 (Vue.js 3 + FastAPI)

Студент создаёт приложение для выбора и настройки моделей машинного обучения: загружает данные, выбирает модель, настраивает гиперпараметры через динамические формы, запускает обучение и отображает метрики. Для выполнения работы требуется реализация бэкенда на FastAPI с классом MLService. Преподавателю рекомендуется обратить внимание на корректную настройку прокси в Vite и обработку CORS на сервере.

## 5. Воркшоп: формат и представление результатов

Воркшоп **«Визуализация графиков для ML моделей»** проводится в формате командной работы. Студенты делятся на две команды: одна использует Chart.js, другая — ECharts. Продолжительность воркшопа — 4 академических часа.

**Задания воркшопа:**

1. Построение столбчатой диаграммы (разогрев).
2. Визуализация результатов линейной регрессии (scatter plot).
3. Визуализация кластеризации K-Means с отображением центроидов.
4. Сравнение трёх моделей на одном графике.
5. Разработка финального дашборда с тремя графиками.

**Представление результатов:**
По завершении воркшопа каждая команда готовит устную презентацию продолжительностью 10–15 минут, в которой:
- демонстрирует работающие графики и дашборд;
- объясняет ключевые архитектурные решения (структура компонентов, управление состоянием, обработка данных);
- проводит сравнительный анализ производительности, удобства настройки и интерактивных возможностей своей библиотеки.

Преподаватель оценивает как техническую реализацию, так и качество представления. Вопросы командам задаются после каждой презентации. Формат воркшопа позволяет студентам освоить две альтернативные библиотеки визуализации, что формирует навык обоснованного выбора инструмента под конкретную задачу.

## 6. Организация самостоятельной работы студентов

Часть заданий лабораторных работ и воркшопа может выполняться студентами во внеаудиторное время в рамках самостоятельной работы.

**Рекомендуемое распределение самостоятельной работы:**

- **Лабораторная работа №1:** задания 1–8 (разметка и формы) выполняются в аудитории, задания 9–10 (стилизация и интерактивные состояния) — дома.
- **Лабораторная работа №2:** задания 1–6 (базовые операции с DOM) — в аудитории, задания 7–10 (загрузка данных, визуализация, комплексное приложение) — дома.
- **Лабораторная работа №3:** этапы 1–2 (настройка проекта, компоненты) — в аудитории, этапы 3–4 (бэкенд, тестирование) — дома.
- **Воркшоп:** задания 1–3 (первые графики, линейная регрессия) — в аудитории, задания 4–5 (сравнение моделей, дашборд) могут быть завершены дома.

Для домашней работы студенту необходимо наличие компьютера с установленным программным обеспечением, доступом к интернету и исходным файлам (CSV, JSON). Преподавателю рекомендуется предоставить студентам все необходимые данные в начале изучения модуля, а также организовать канал оперативной связи (чат или форум) для решения технических вопросов при выполнении работ дома.
