# Методические указания для преподавателей по оцениванию модуля «Frontend»

## 1. Общие положения

Настоящие методические указания предназначены для преподавателей, осуществляющих оценивание результатов освоения модуля «Frontend» в рамках дисциплины «Технологии развертывания моделей машинного обучения в клиент-серверных приложениях». Модуль включает три вида аттестационных мероприятий: тестирование, лабораторные работы и воркшоп. Критерии оценивания каждого мероприятия зафиксированы в соответствующих учебно-методических материалах. В настоящих указаниях приведены шкалы перевода баллов в оценки и сводная таблица вклада каждой части в итоговую оценку по модулю.

## 3. Оценивание лабораторных работ

Лабораторные работы выполняются студентами индивидуально. В рамках модуля предусмотрены три лабораторные работы, выполняемые последовательно. Критерии оценивания для лабораторных работ по HTML/CSS и JavaScript объединены в общую шкалу. Лабораторная работа по Vue.js 3 и FastAPI оценивается отдельно.

### 3.1. Лабораторная работа «Визуализация данных в ML моделях. Работа с HTML и CSS» и лабораторная работа «Визуализация данных в ML моделях. JavaScript»

Оценивание производится по пяти критериям. Максимальный общий балл за обе работы — 10.

| Критерий | Баллы |
|----------|-------|
| HTML и CSS — создана семантически корректная структура HTML-документа с заголовками, абзацами, списками, гиперссылками и изображениями; реализованы формы с текстовыми полями, чекбоксами, радиокнопками, выпадающими списками и полем загрузки файлов; все поля имеют корректные подписи и атрибуты; применены CSS-стили для оформления страницы и интерактивных состояний элементов | 2 |
| JavaScript (базовый) — реализованы операции с DOM-элементами, обработка событий клика, фокуса, изменения, валидация полей ввода, переключение видимости пароля, загрузка текстовых файлов через FileReader | 2 |

| Качество кода — код структурирован, разделен на компоненты и сервисы, присутствуют комментарии, соблюдены соглашения по именованию | 1 |

**Порядок защиты лабораторных работ:**

1. Студент демонстрирует работающее приложение в браузере.
2. Преподаватель проверяет выполнение каждого задания, задавая уточняющие вопросы по коду.
3. Студент должен объяснить ключевые архитектурные решения и обосновать выбор используемых подходов.

## 4. Оценивание воркшопа

Воркшоп **«Визуализация графиков для ML моделей: ECharts vs Chart.js»** проводится в формате командной работы. Оценивание осуществляется по шести критериям. Максимальный суммарный балл — 15. Оценка выставляется каждой команде.

| Критерий | Баллы |
|----------|-------|
| Загрузка данных из CSV/JSON | 2 |
| Работающий график (любой) | 2 |
| Интерактивность (tooltip, zoom) | 2 |
| ML модель (обучение + предсказание) | 4 |
| Сравнение 2+ моделей | 3 |
| Финальный дашборд | 2 |
| **ИТОГО** | **15** |

### 4.2. Процедура оценивания

1. **Устная презентация (5–10 минут):** каждая команда демонстрирует работающее приложение, объясняет архитектурные решения и проводит сравнительный анализ своей библиотеки с альтернативной.
2. **Демонстрация кода:** преподаватель задаёт уточняющие вопросы по ключевым фрагментам кода (инициализация графика, настройка опций, обучение модели, управление состоянием).
3. **Вопросы к команде:** оценивается способность команды аргументировать выбор библиотеки, обосновывать архитектурные решения и предлагать улучшения.
